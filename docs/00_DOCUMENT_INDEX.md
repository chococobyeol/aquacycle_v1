## 문서 우선순위

내용이 충돌하면 다음 순서로 판단한다.

1. 현재 실행된 코드와 최신 자동·수동 검증 결과
2. `02_IMPLEMENTATION_SPEC.md`의 현재 구현 규칙
3. `07_CONTENT_ROADMAP.md`와 연결된 상세 콘텐츠 명세의 아직 구현되지 않은 규칙
4. `04_SIMULATION_MODEL.md`와 05~06의 모델·하위 시스템 기준
5. `01_GAME_DESIGN.md`의 장기 기획과 과거 논의 기록
6. `03_TECHNICAL_ARCHITECTURE.md`의 초기 기술 선택 근거

기획 문서의 과거 초안이 현재 구현 명세와 다르면 현재 구현 명세를 따른다. 다음 콘텐츠가 구현 완료되면 확정된 내용을 02·04·08에 옮기고, 07에는 아직 남은 작업만 유지한다.

## 새 문서를 추가하는 규칙

- 기존 문서의 역할에 들어가는 내용은 새 파일을 만들지 않고 해당 문서에 합친다.
- 독립된 하위 시스템 기록이 반드시 필요할 때만 새 번호를 추가한다.
- 미션별 상세 기획은 `docs/content/` 아래에 `MISSIONS_07_08_<TOPIC>.md` 형식으로 둔다.
- 파일명은 영문 대문자 스네이크 표기와 역할 이름을 사용한다.
- 파일명에 날짜, `FINAL`, `NEW`, `LATEST`, `V3` 같은 임시 표기를 붙이지 않는다.
- 모든 새 문서는 이 안내 문서와 관련 문서의 상단 링크에 추가한다.
