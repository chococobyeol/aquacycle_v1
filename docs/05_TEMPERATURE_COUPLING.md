# 05. 수온–생태 연결 구현과 검증

> 갱신일: 2026-07-23
> 적용 상태: 배포본 기반 모델에 구현 완료, 장기 회귀 통과
> 범위: 닫힌 공기층 산소 분배, 체리새우, 분해균, 질산화군집, 관찰·사망 기록

## 1. 왜 이번 연결이 구조적인가

공간 수온만 보이고 생물·산소가 반응하지 않으면 수온은 장식값에 불과하다. 반대로 생물마다 별도 온도장을 만들면 이후 물고기·수초를 추가할 때 물리와 생태 규칙이 갈라진다. 이번 구현은 기존 `36×20` 공통 물 수송 격자를 그대로 물리 host로 두고, 모든 생태 반응이 자기 위치의 수온을 읽게 했다.

```text
공통 물리 host
├─ 수온·속도·압력·구조물 저항
├─ 산소·무기탄소·유기물·암모니아·영양염 수송
└─ 수면 기체 교환

생태 반응 모듈
├─ 조류: 기존 종별 수온 곡선
├─ 새우: 대사 / 성장·번식 / 건강 곡선
└─ 균 필름: 먹이·산소 Monod 반응 × theta 수온 보정
```

이 구조는 AED2가 생태 라이브러리를 물리 host에 연결하는 방식과 같은 계층 분리를 따른다. AED2 자체를 복사하거나 의존성으로 넣지는 않았다.

## 3. 구현한 식과 게임 보정

### 3.1 산소 용해도와 닫힌 공기층

`src/simulation/gasExchange.ts`는 USGS가 채택한 Benson–Krause 식을 그대로 계산해 실제 담수 포화값 `mg/L`을 얻는다. 게임 산소값은 실제 `mg/L`가 아니므로 24°C 값을 기준으로 상대화한다.

```text
S(T) = DOsat(T) / DOsat(24°C)
Owater,eq = (Owater + Oair) × S(T) / (1 + S(T))
```

- 24°C에서는 기존 1:1 분배와 같다.
- 차가운 물은 물쪽 산소 몫이 커지고 따뜻한 물은 작아진다.
- 이동한 양만큼 반대 저장고에서 빼므로 총산소는 보존된다.
- 전체 무기탄소는 용존 CO₂만이 아니므로 pH·탄산평형 없이 CO₂ Henry 상수를 억지로 적용하지 않았다.

### 3.2 체리새우

`src/simulation/config.ts`의 동물 정의가 다음 세 반응을 데이터로 가진다.

1. 대사: `1.07^(T-24)`, 안전 제한 `0.55~1.65`
2. 성장·번식: 구간 선형 종 곡선, 33°C에서 0
3. 건강: 20~28°C는 적합도 1, 극단 수온에서 누적 피해

대사 보정은 허기 에너지 감소와 실제 몸·저장 생체량의 유지 호흡 양쪽에 똑같이 적용한다. 성장, 번식 휴지기, 교미, 포란은 별도 번식 곡선을 사용한다. 따라서 높은 수온에서 먹이와 산소 요구는 늘지만 33°C에서는 대사가 사라지는 것이 아니라 번식만 멈춘다.

사망 기록에는 당시 수온과 `temperature` 원인을 남긴다. 선택한 살아 있는 개체에는 현재 수온, 대사 배율, 성장·번식 배율, 생존 적합도를 표시한다.

### 3.3 두 균 기능군

기존 이중 Monod 활동도는 유지한다.

```text
activity = food/(Kfood+food) × oxygen/(Ko2+oxygen)
temperatureFactor = 1.08^(T-24)
uptake = biomass × maximumUptake × activity × temperatureFactor × dt
decay = biomass × decayRate × temperatureFactor × dt
```

증식뿐 아니라 유지·기아 사멸도 같은 시간 보정을 받아, 차가운 곳에서 군집의 전체 반응 시계가 느려지고 따뜻한 곳에서 빨라진다. 먹이와 산소가 부족하면 수온이 높아도 순성장은 음수다. 탐침의 예상 순성장률과 선택 표면의 수온 배율에도 같은 함수를 사용해 화면 설명과 실제 계산이 어긋나지 않게 했다.

## 4. 일부러 넣지 않은 것

- 수온 의존 성비: 작은 표시 개체군의 우연한 단절을 크게 만들고, 이번에 채택한 생존·번식 연구만으로 안정적인 규칙을 확정하기 어렵다.
- 33°C 즉사: 연구가 보여 준 것은 번식 억제이며 즉사 문턱이 아니다.
- 전체 무기탄소의 Henry 분배: pH·알칼리도·CO₂/HCO₃⁻ 분리를 하지 않은 압축 탄소 저장고에 적용하면 오히려 화학적으로 틀린다.
- 물고기·수초 전용 조건문: 이후 종 정의 데이터로 추가할 수 있도록 공통 보간·theta 함수만 만들었다.
- 별도 온도 격자나 유체 엔진: 이미 검증한 공통 물 수송 격자를 재사용한다.

## 5. 검증 기록

### 5.1 수치·단위 테스트

`tests/temperatureCoupling.test.ts`

- Benson–Krause 대표값: 0°C `14.62`, 20°C `9.09`, 30°C `7.56 mg/L`
- 10°C 물의 평형 산소 몫이 30°C보다 큼
- 물–공기층 교환 전후 총산소 오차 `< 0.0001`
- 같은 먹이·산소에서 28°C의 두 균 순성장과 실제 1초 성장량이 16°C보다 큼
- 새우 대사와 번식 곡선이 분리되어 있고 33°C 번식 계수가 0

### 5.2 기존 미션·물질 회귀

- 전체 자동 회귀: `39개 파일, 183개 테스트` 통과
- 수온 연결 집중 검사: `tests/temperatureCoupling.test.ts`의 `6개 테스트` 통과
- 2시간 폐쇄 수조: 새우 여러 세대 생존, 두 균 유지, 저산소·독성사 없음
- 마지막 30분 무기탄소 범위 `23.08~27.89`, 한 방향 소실이 아니라 생산자–소비자 위상에 따른 반등 확인
- 질소·탄소·산소 등가 장부 상대 오차 각각 `< 0.00000001%`
- 평균 수온 `24.11°C`, 최고 수온 `< 31°C`
- 64배속 30분 성능 회귀에서 네 수질장과 두 균 군집이 유한 범위를 유지
- 전체 수질 순환이 꺼진 미션 4에서도 극단 수온 피해가 에너지 표시값에 덮어써지지 않고 실제 건강에 누적됨

끝점 두 개가 가까운지만 보는 기존 검사는 진동의 반대 위상을 불안정으로 오판할 수 있어, 마지막 구간의 상·하한과 상승·하강 존재를 함께 검증하도록 바꿨다. 질량보존과 생존 안전 기준은 완화하지 않았다.

### 5.3 실제 화면·패키지 검증

- `1280×720` 화면에서 미션 5 관찰 지도를 열어 수질·미생물과 기체·물질 장부를 동시에 전개했다.
- 수면 수온, 1기압 담수 산소 포화도, 현재 물쪽 산소 평형 목표가 실제 계산값으로 표시됨을 확인했다.
- 고정 패널의 화면 밖 넘침 `0건`, 브라우저 콘솔 경고·오류 `0건`을 확인했다.
- `npm run typecheck`, `git diff --check`, `npm run package`를 통과했다.
- macOS arm64 독립 패키지의 main·preload·renderer production bundle 생성과 Electron 패키징을 완료했다.

## 6. 이후 종 추가 규칙

새 동물·수초·균 기능군을 추가할 때 물리 코드를 수정하지 않는다.

- 동물: 정의 데이터에 `referenceTemperature`, `metabolicTheta`, 번식·건강 곡선 추가
- 광합성 생물: 기존 빛·수온 곡선과 유한 질소·탄소 장부 사용
- 균 기능군: 먹이 변수, Monod 반포화값, 수율, 산소 요구, theta 정의
- 관찰: 실제 계산에 사용한 같은 함수의 결과를 수치와 곡선으로 공개

이 규칙을 지키면 히터·냉각기·낮밤은 경계 열원 오브젝트로만 추가되고, 기존 생물 반응과 산소 용해도는 구조 변경 없이 자동으로 반응한다.
