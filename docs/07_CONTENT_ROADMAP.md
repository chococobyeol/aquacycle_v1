# 07. 콘텐츠 로드맵

> 갱신일: 2026-07-24
> 목적: 전체 미션의 순서, 핵심 관계, 구현 상태와 상세 명세 위치를 한눈에 관리한다.

이 문서는 개별 생물의 수식이나 긴 구현 설명을 담지 않는다. 전체 콘텐츠의 진행 상황과 상세 문서로 가는 링크만 유지한다. 각 미션 묶음의 자세한 생태 규칙, 지급 자원, 성공 판정과 검증 조건은 `docs/content/` 아래의 상세 명세에 기록한다.

미션의 번호는 구현 순서다. 플레이어가 앞 미션을 완료해야 다음 미션을 시작할 수 있다는 뜻은 아니다. 현재 원칙대로 모든 미션을 목록에서 바로 열 수 있고 완료 여부만 기록한다.

## 콘텐츠 묶음

### 1–3 · 생산자와 빛

- 위치별 광량
- 구조물 표면과 조류 정착
- 종별 광량 적합도

### 4–6 · 소비자와 물질 순환

- 체리새우의 실제 섭식과 개체군
- 분해·질산화와 닫힌 물질 장부
- 낮·밤의 생산자 호흡과 산소 변화

### 7–8 · 포식자와 부유 먹이망

- 송사리의 국소 사냥과 피난처
- 알·치어·성체 생애 주기
- 실제 크기가 커진 수조
- 부유 미세조류–물벼룩–송사리 먹이망

상세 내용은 [미션 07–08 상세 명세](./content/MISSIONS_07_08_RICEFISH_DAPHNIA.md)를 따른다.

## 미션 9 이후 추가 규칙

새 미션을 논의할 때 이 문서에 먼저 한 줄을 추가하고 상태를 `논의 중`으로 표시한다. 핵심 관계와 앞 미션과의 차이가 정리된 뒤에만 상세 명세를 만든다.

상세 문서 파일명:

```text
docs/content/MISSIONS_09_10_<TOPIC>.md
docs/content/MISSIONS_11_12_<TOPIC>.md
```

- 미션 번호는 두 자리로 맞춘다.
- `<TOPIC>`은 생물 이름을 나열하기보다 해당 묶음의 핵심 관계를 나타낸다.
- 한 미션만 독립적으로 길어지면 `MISSION_09_<TOPIC>.md`처럼 단일 번호를 사용한다.
- 아직 이름과 역할이 정해지지 않은 미션을 빈 파일로 미리 만들지 않는다.
- 구현이 끝나면 확정 규칙을 `02_IMPLEMENTATION_SPEC.md`와 `04_SIMULATION_MODEL.md`에 반영하고 이 표의 상태를 `구현·검증 완료`로 바꾼다.
- 배포 검증 결과는 `08_RELEASE_VERIFICATION.md`에 기록한다.

| 상태 | 의미 |

| 논의 중 | 핵심 관계나 범위가 아직 바뀔 수 있음 |

| 구현·검증 완료 | 현재 배포 검증 기준을 통과함 |
