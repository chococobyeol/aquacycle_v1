# 배포본 검증 계약과 실행 기록

> 갱신일: 2026-07-24
> 대상: 현재 `main` 작업본의 미션 1~6, 실험실, 생태·물질 순환 및 관찰 기능

이 문서는 과거 어느 시점의 통과 개수만 남기는 기록이 아니라, 현재 배포본에서
무엇을 실제 문제로 판정하는지와 같은 검사를 다시 실행하는 방법을 함께 고정한다.

## 1. 배포 전 일괄 검증

```text
npm run verify:release
```

이 명령은 다음 순서로 실행된다.

1. TypeScript 정적 검사
2. 전체 `240`개 자동 회귀를 단일 작업자에서 실행
3. 미션 6 생산자·미생물계의 낮·밤 50주기 확장 검증

단일 작업자는 모델을 느슨하게 만드는 설정이 아니다. 서로 독립적인 장시간
시뮬레이션들이 같은 CPU를 동시에 차지해 성능 제한시간만 넘기는 거짓 실패를
막는다. 64배속 계산량과 Worker 성능 계약은 각각의 전용 시험에서 그대로 검사한다.

## 2. 생태 검증 작성 규칙

- 과제 해법 검증은 목록에서 항목을 고르고 수조에 놓는 공개 명령만 사용한다.
- 정상 플레이에서 만들 수 없는 방식으로 모든 셀의 생체량을 직접 채우지 않는다.
- 광원 출력 숫자가 크다는 이유만으로 생산성이 높다고 가정하지 않는다. 종별
  광반응을 지난 실제 생산량·생체량·누적 섭식량을 관찰한다.
- 순간 개체 수 하나로 숨은 수용량을 추정하지 않는다. 출생 기록, 실제 섭식,
  세대교체와 기술 안전선이 생태 판정에 개입하지 않는지를 나눠 확인한다.
- 새우 개체군의 멸종과 탄소·질소·산소 등가 장부 누출은 별개로 판정한다.
- 미션 해법은 특정 장식 배치를 강요하지 않되, 공급된 먹이를 볼 수 있는 위치에
  배치하는 것처럼 플레이어가 화면에서 판단할 수 있는 행동으로 구성한다.

단위 수식 시험은 필요한 상태 하나만 고립하기 위해 내부 값을 설정할 수 있다.
그러나 그 결과를 그대로 과제의 플레이 가능성이나 전체 생태 평형의 증거로 쓰지
않는다.

조작 불능 문제는 없었다.
