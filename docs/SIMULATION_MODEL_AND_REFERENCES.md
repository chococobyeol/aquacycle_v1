> 관련 문서: [핵심 플레이 기획](./CORE_GAMEPLAY.md) · [V2 재설계 명세](./V2_REDESIGN.md) · [초기 기술 구조](./MVP_TECHNICAL_ARCHITECTURE.md) · [수온 생태 연결 구현·검증](./TEMPERATURE_COUPLING_IMPLEMENTATION.md) · [화학량론 장부 구현·검증](./STOICHIOMETRIC_LEDGER_IMPLEMENTATION.md)

이 문서는 AquaCycle의 생태 규칙이 어떤 현실의 관계를 보존하고, 어떤 부분을 게임에 맞게 압축했는지 추적하기 위한 기술 근거 문서다. 공간 수온과 공통 물 수송까지 현재 코드에 구현된 모델과 앞으로 보정할 항목을 구분한다.

## 1. 표기와 출처 사용 원칙

각 규칙은 다음 네 종류 중 하나로 표시한다.

- **관찰 근거**: 논문·공공기관·공식 모델 문서에서 확인되는 관계
- **현재 구현**: 2026-07-22 현재 플레이어블 코드에 존재하는 동작
- **후속 설계**: 현재 범위 다음에 검토하거나 보정할 동작
- **게임 보정**: 실제 단위나 시간을 그대로 재현하지 않고 플레이 가능한 속도와 범위로 조정한 값

출처는 모델의 **관계와 상대적 경향**을 뒷받침한다. 논문에 등장하는 종·계통·사육 조건이 게임 수조와 다르면 해당 논문의 수치를 그대로 복사하지 않는다. 게임 상수는 자동 시뮬레이션과 플레이 테스트로 보정하며, 실측 단위를 표방하지 않는 값에는 실제 단위를 붙이지 않는다.

예를 들어 체리새우의 야외 수명이 약 10~15개월이라는 연구는 자연사와 세대교체를 구현할 근거다. 현재 게임 수명 `900~1350` 시뮬레이션초는 그 기간을 그대로 환산한 값이 아니라 한 번의 실험에서 여러 세대를 볼 수 있게 한 시간 압축이다.

| 대상 | 현재 계산 단위 | 비고 |
|---|---|---|
| 구조물·바닥 생산자 | 표면 셀별 두 조류와 나사말 생체량 | 각 종 `0~1`, 같은 표면 공간 공유. 나사말은 바닥에만 정착 |
| 체리새우 | 암수·나이·에너지·몸 질량을 가진 개별 개체 | 이동은 연속 2D 좌표 |

| 물속 무기탄소 | `36×20` 농도장 | 다른 용존 성분과 같은 물 흐름 사용 |
| 공기층 CO₂·O₂ | 기체별 저장고 값 하나 | 수면을 통해 물과 교환 |
| 수온·물 흐름 | `36×20` 열에너지장과 2D 속도장 | 광원·경계 열교환·구조물·부력 반영 |

### 2.2 현재 계산 순서

기본 생태 스텝은 `0.25` 시뮬레이션초이며, 32배속 이상에서는 `1`초로 묶는다.

```text
공간 열전도·경계 열교환·부력 유동
→ 낮·밤 광량장 갱신 → 생산자 총광합성·호흡·스트레스·경쟁·표면 전파
→ 새우 섭식·에너지·성장·번식·사망
→ 균 필름의 먹이·산소 반응
→ 다섯 용존 성분의 국소 확산과 공통 유량 수송
→ 닫힌 공기층과 기체 교환
→ 미션 판정과 관찰 스냅샷
```

기존 `bulkMixing`은 제거했다. 온도 차이에서 계산한 하나의 속도·경계 유량을 유기물·암모니아성 노폐물·영양염·용존산소·무기탄소가 함께 사용한다. 따라서 먼 두 위치가 평균 혼합 항으로 직접 가까워지지 않고 실제 격자 경계를 거쳐 이동한다.

### 2.3 현재 공간 수송 보정값

아래 값은 실제 확산계수가 아니라 `초⁻¹` 단위의 게임 보정률이다. 공통 방향성 유동이 해상하지 못하는 분자·소규모 혼합만 나타낸다.

| 장 | 국소 이웃 확산 |
|---|---:|
| 용존 유기물 | 0.045 |
| 암모니아성 독성 노폐물 | 0.09 |
| 영양염 | 0.07 |
| 용존산소 | 0.12 |
| 물속 무기탄소 | 0.10 |

국소 확산과 공통 경계 유량은 모두 한 셀이 잃은 양을 다른 셀에 그대로 더한다. 양수성과 셀 용량을 지키는 송신·수신 제한자를 사용하며, 수송만으로 총량을 바꾸지 않는다.

## 3. 현재 공통 열·물질 수송 모델

### 3.1 목표

현재 모델은 산업용 CFD의 정확한 난류를 재현하는 것이 아니라, 다음 질문에 공간적으로 납득할 수 있는 답을 제공한다.

- 왜 이 위치의 물이 더 따뜻하거나 차가운가?
- 왜 표면의 산소가 아래쪽으로 전달되었는가?
- 왜 사체 주변의 유기물과 암모니아가 이 방향으로 퍼졌는가?
- 왜 돌 뒤쪽과 흐름이 약한 곳에 물질이 오래 남는가?
- 왜 조명·유리벽·바닥 배치가 수온과 수질의 경로를 함께 바꾸는가?

이를 위해 다섯 용존 성분과 공간 수온을 **같은 물 흐름을 공유하는 보존형 수송 계층**으로 통합했다.

### 3.2 모델 수준

채택한 수준은 `36×20` 저해상도 2D 부력 유동장이다.

```text
각 물 셀
├─ 온도 또는 열에너지
├─ 가로·세로 유속
├─ 물 비율·구조물 점유율·통과 저항
├─ 용존산소
├─ 물속 무기탄소
├─ 용존 유기물
├─ 암모니아성 독성 노폐물
└─ 영양염
```

고형 찌꺼기, 표면 조류와 균 필름은 이 배열에 합치지 않는다. 고형 찌꺼기는 침전·부유 상태에 따라 별도 이동하고, 조류와 균은 구조물 표면에 남으면서 인접한 물 셀과 반응한다.

### 3.3 유지할 물리 관계

온도와 각 용존 성분은 다음의 이류–확산–반응 구조를 따른다.

```text
온도 변화
= 물 흐름이 운반한 열
+ 주변 물질과의 열전도
+ 흡수한 광원 에너지
+ 외부 공기와의 열교환

용존 성분 변화
= 물 흐름이 운반한 질량
+ 성분별 분자·미해상 혼합
+ 생물학적 생성과 소비
```

수학적으로는 다음 관계를 저해상도로 이산화한다.

```text
∂T/∂t + u·∇T = α∇²T + Q/(ρcp)
∂Ci/∂t + ∇·(u Ci) = Di∇²Ci + Ri
```

- `T`: 온도
- `u`: 공통 물의 속도
- `Ci`: 산소·탄소·유기물·노폐물·영양염 각각의 농도
- `α`, `Di`: 열과 성분별 확산 계수
- `Q`: 조명·외부 경계에서 들어오거나 나가는 열
- `Ri`: 광합성·호흡·분해·질산화 같은 반응

### 3.4 자연 대류

물의 온도 차이가 작다는 범위에서는 Boussinesq 근사를 사용한다. 밀도를 모든 계산에서 바꾸지 않고 온도 차이가 만드는 부력에만 반영한다.

```text
세로 부력 = 부력계수 × (현재 온도 - 기준 온도)
```

그 결과 다음 관계가 나온다.

- 아래에서 데워진 물은 상승한다.
- 유리벽이나 수면에서 식은 물은 하강한다.
- 위쪽만 따뜻한 층은 비교적 안정되어 대류가 약해진다.
- 국소 가열과 측면 냉각이 함께 있으면 완만한 순환이 생긴다.

유속이 한 셀에 쌓여 물이 생성·소멸하는 것처럼 보이지 않도록 `0.5` 시뮬레이션초마다 압력 투영을 수행한다. 현재 `36×20` 격자는 Jacobi 반복 `14`회를 사용한다. Jos Stam의 실시간 유체 방법은 작은 격자에서 속도와 압력 투영을 구성하는 구현 참고로 사용하되, 물질 장부에는 질량 손실이 생길 수 있는 단순 반라그랑주 스칼라 이류를 그대로 사용하지 않는다.

### 3.5 보존형 물질 이동

폐쇄 수조에서 수치 오차로 산소·질소·탄소가 사라지지 않도록 셀 중심값을 임의로 보간해 덮어쓰지 않는다. 셀 경계마다 물의 체적 유량을 한 번 계산하고, 같은 유량으로 모든 용존 성분을 옮긴다.

```text
경계 유량 F가 A → B라면

A의 감소량 = F × A의 농도 × dt
B의 증가량 = A의 감소량
```

내부 경계에서는 한쪽이 잃은 양이 다른 쪽에 정확히 더해져야 한다. 광합성·섭식·호흡·분해처럼 형태가 바뀌는 반응도 기존 C:N 장부에서 출발 물질과 도착 물질을 함께 기록한다. 외부와 실제로 교환되는 열과 기체만 별도 경계 장부에 기록한다.

현재 구현은 양수성과 보존이 명확한 1차 상류 유한체적 수송을 사용한다. 수치 확산이 군락·수질 지도를 지나치게 흐리면 이후 제한자 기반 2차 수송을 검토한다. MITgcm과 MOM6의 추적자 수송 문서는 흐름과 일치하는 경계 flux, 단조성 및 물질 보존을 유지하는 참고 기준으로 사용한다.

### 3.6 전도와 복사열

열전도는 물과 물 사이뿐 아니라 물·돌·바닥·유리 사이에서 처리한다.

```text
전도 열량 = 경계 유효 열전도도 × (이웃 온도 - 현재 온도) × dt
```

각 구조물의 보이는 외곽을 셀당 `3×3` 표본으로 `36×20` 격자에 부분 점유율로 투영하고, 재질별 열용량과 열전도도를 부여한다. 돌 전체를 온도 하나로 즉시 균일화하지 않고 여러 격자 구간이 천천히 열을 전달하게 한다.

복사열은 모든 셀 사이의 복사 교환을 풀지 않는다. 현재 광량장이 계산한 직접광·가림·산란·반사광을 에너지 입력으로 재사용한다.

```text
흡수열 = 광량 × 물질별 흡수율 × dt
```

물, 돌, 바닥은 서로 다른 비율로 빛을 흡수한다. 빛을 받은 구조물이 먼저 데워지고 인접한 물로 전도할 수 있어야 한다. 수면과 유리벽은 실내 온도와 Newton형 경계 열교환을 하며, 내부 전도와 대류는 열을 재배치할 뿐 새 열을 만들지 않는다.

### 3.7 2D 구조물의 보정

화면의 돌을 완전 불투과 벽으로 바꾸면 현실에서는 앞뒤로 지나갈 물까지 막히는 문제가 생긴다. 구조물은 다음 두 값을 구분한다.

- `solidFraction`: 열용량과 열전도에 사용되는 실제 고체 점유율
- `permeability` 또는 `drag`: 3차원 앞뒤 우회를 압축한 물 통과 저항

돌 중심은 흐름이 매우 약하고 가장자리는 일부가 통과하도록 한다. 이 방식은 구조물 뒤의 느린 영역은 만들되, 2D 화면에서 밀폐되지 않은 틈이 영구적인 무산소 포켓으로 오판되는 것을 줄인다.

### 3.8 계산 순서

```text
1. 조명·실내 공기·생물 반응에서 열과 물질 source/sink 기록
2. 열전도와 성분별 국소 확산
3. 온도 차이에서 부력 추가
4. 유속 이류·저항·점성 적용
5. 압력 투영으로 발산 억제
6. 셀 경계 물 유량 계산
7. 같은 경계 유량으로 열과 모든 용존 성분 보존형 이동
8. 수면 기체 교환과 경계 열교환
9. 음수·비유한값·장부 오차 검증
```

반응과 수송의 순서가 결과를 크게 바꾸지 않도록 필요하면 반응을 반 스텝씩 나누는 연산자 분할을 사용한다. 32·64배속의 `1초` 생태 스텝에서 경계 유량이 한 셀 전체를 초과하면 1~4회의 수송 하위 스텝을 사용한다.

### 3.9 예상 성능

격자는 720셀이다. 속도 2개, 압력·발산 임시 배열, 온도와 다섯 용존 성분을 추가해도 메모리는 매우 작다. 비용은 현재 네 필드의 확산보다 증가하지만, 모든 성분이 같은 경계 유량과 장애물 정보를 재사용하므로 변수 하나를 추가할 때마다 유체 전체를 다시 풀지는 않는다.

현재 구현 기준은 다음과 같다.

- Web Worker의 생태 스텝에서만 계산하고 렌더 프레임마다 풀지 않는다.
- `Float32Array`와 재사용 scratch 배열을 사용한다.
- 구조물이 움직이거나 바뀔 때만 고체 점유·저항 지도를 다시 만든다.
- 고배속에서도 압력 반복 `14`회, 열 하위 스텝 최대 `4`회라는 명시적 상한을 둔다.
- 64배속 30분 회귀는 개발 환경에서 약 `6.2`초, 2시간 폐쇄 생태계 회귀는 약 `35.4`초에 완료됐다.

## 4. 조류 모델과 근거

### 4.1 유지하는 현실 관계

**관찰 근거**는 다음과 같다.

- 광합성 생물의 빛 반응은 일반적으로 저광량 제한, 포화, 경우에 따라 강광 저해를 가진다.
- 같은 속의 조류도 계통과 순응 조건에 따라 최적 광량과 수온이 다를 수 있다.
- 붓뚜껑말속은 사상형 녹조류로, 수조 화면에서는 개별 세포보다 표면 군락의 질감으로 보는 편이 맞다.
- `Nitzschia palea`는 저광·고광 조건에 광순응할 수 있으며, 종 전체를 “밝으면 즉사하는 규조류”로 정의할 근거는 없다.
- 부착 규조류와 다른 미생물은 표면 생물막을 이루며 빛·영양염·섭식의 영향을 함께 받는다.

### 4.2 현재 구현

두 부착 조류와 나사말은 각 표면 셀의 `0~1` 생체량으로 계산한다. 나사말은 바닥 셀을 뿌리점으로 사용하는 개별 포기(ramet)로 추가 추적한다. 포기는 나이·개체별 수명·느리게 변하는 잎 구조 크기·건강·러너 준비도·부모 포기를 가진다. 밤의 호흡으로 저장 생체량이 조금 줄더라도 잎 길이가 매일 수축하지 않지만, 장기 자원 부족과 노쇠에는 잎이 실제로 줄고 변색된다.

지급 포기는 이미 뿌리내린 어린 포기로 시작해 `360`초에 성체가 된다. 개체별 압축 수명은 결정적 난수로 `2,400~3,300`초이며 수명의 `82%`부터 노쇠 단계에 들어간다. 건강한 성체는 약 `600`초의 환경 적합 누적마다 가까운 빈 바닥 셀을 찾아 러너 자손을 만든다. 이때 부모의 실제 생체량 `0.16`을 딸 포기로 옮기므로 번식 자체가 물질을 복제하지 않는다. 적합한 빈 바닥이 없으면 준비도만 유지하고 강제로 번식하지 않는다. 숨은 최대 포기 수는 두지 않으며 바닥 공간·광량·수온·무기질소·무기탄소와 부모 저장량이 확장을 제한한다.

생체량 `0.055` 미만 상태가 `150`초 지속되거나 개체별 수명에 도달하면 포기는 죽는다. 남은 생체량 전부를 기존 조류 자연 소실과 같은 찌꺼기 경로로 보내 분해균·질산화균·영양염·무기탄소 순환에 연결한다. 사용자가 심은 포기와 러너 자손은 저장 파일에 나이·수명·잎 구조·러너 진행도를 함께 기록한다. 관찰창에서는 어린 포기/성체/노쇠 수, 선택 포기의 나이·건강·러너 준비도와 자손 수를 표시한다.

```text
기준 순성장률 = 종별 구간 선형 광량 곡선
수온 적합도 = 종별 구간 선형 수온 곡선

기준 총광합성 = max(0, 기준 순성장률 + 24°C 기준 호흡률)
총광합성 = 기준 총광합성 × 수온 적합도
호흡 = 24°C 기준 호흡률 × theta^(수온-24)
광·수온 스트레스 = 기준 비호흡 손실 + 부적합 수온 손실
실제 새 생체량 = 기존량 × 총광합성 × 빈 표면 × 영양·탄소 제한 × dt
순변화 = 실제 새 생체량 - 호흡 - 광·수온 스트레스 - 자연 소실
```

기존 순성장 곡선은 24°C에서 같은 결과를 내도록 총광합성·호흡·비호흡 손실로 분해했다. 따라서 낮에는 총광합성이 무기질소와 무기탄소를 실제로 고정하고 산소를 만들며, 어두울 때는 총광합성이 0이 되어도 호흡이 계속되어 산소를 소비하고 무기탄소·암모니아성 질소를 돌려놓는다. 산소가 바닥나 요청된 호흡을 모두 수행하지 못한 생체량과 광·수온 스트레스 손실은 고형 찌꺼기로 돌아간다. 생리 함수는 어둠의 원인을 받지 않는다. 같은 국소 광량·수온·자원이라면 밤, 구조물 그림자, 꺼진 전등에서 모두 같은 반응을 반환한다. 광순응을 이후 도입한다면 시나리오 구분이 아니라 각 군락이 실제로 겪은 누적 광량 이력으로 계산한다.

미션 6의 주기는 `새벽 30초 → 낮 240초 → 해질녘 30초 → 밤 60초`이며 **수면 전체에 들어오는 자연광**이 낮의 `100%`에서 밤의 `4.5%`까지 완만하게 변한다. 숨은 천장 전등은 사용하지 않는다. 자연광과 전등은 별도 성장 규칙이 아니라 위치·폭·방향·시간 배율이 다른 공통 광원 자료로 만들고, 동일한 구조물 차폐·수심 감쇠·산란·1차 반사 계산에 넣은 뒤 한 지점에서 합산한다. 생산자는 광원의 종류가 아닌 이 합산 광량에만 반응한다. 광량 오버레이·표면 광량·수온 입력도 같은 광량장을 사용한다. 고배속에서 매 생태 틱마다 레이캐스팅하지 않도록 새벽·해질녘은 자연광 배율이 4% 이상 달라질 때 광량장을 다시 계산하되 위상 경계는 정확히 갱신한다.

세 생산자는 같은 표면 수용력을 공유하고, 현재 환경에서 상대 순성장률이 높은 종이 서서히 자리를 대체한다. 발달한 붓뚜껑말 군락은 낮은 규조류 막을 추가로 가리는 경쟁 보정을 가진다.

표면 전파는 새 생체량 생성이 아니다. 출발 셀에서 실제 생체량을 차감해 인접·접촉 표면으로 옮기며, 도착지의 빛·수온과 빈 공간이 정착량을 제한한다.

현재 반응곡선의 꼭짓점은 다음과 같다. 두 꼭짓점 사이의 값은 선형 보간한다.

### 4.3 게임 보정과 한계

- 광량 `0~100`은 lux가 아니라 조류 성장에 사용되는 상대적 광합성 유효광량이다.
- 광량·수온 곡선의 정확한 꼭짓점은 미션의 공간 퍼즐과 관찰 가능 시간을 위한 보정값이다.
- 분해되지 않은 유기물이 물을 가득 채워도 광학적으로 아무 영향이 없던 문제를 막기 위해 생산자의 양의 성장에는 `Torg = exp(-0.025 × 유기물)`의 수질 투과율을 곱한다. 이는 유색 용존 유기물과 부유 유기물이 빛을 흡수·산란한다는 관계를 Beer–Lambert형으로 압축한 게임식이며, `0.025`는 실제 농도 단위의 흡광계수가 아니다. 유기물 `0/5/100`에서 투과율은 각각 약 `1.00/0.88/0.08`이고, 물질을 삭제하지 않고 광이 생산자에게 도달하는 비율만 바꾼다.
- `Nitzschia palea`는 실제 모든 계통의 대표가 아니라 **저광량에 순응한 게임 내 수조 계통**으로 명시한다.
- 규조류가 필요로 하는 규산염을 현재 `nutrients`와 별도로 계산하지 않는다. 규조류 장기 제한을 더 깊게 다룰 때 독립 규산염 변수를 검토한다.
- 실제 부착 생물막은 조류·세균·균류·원생생물·고형물이 섞인 복합체지만, 현재 화면과 먹이 계산은 두 조류와 두 균 기능군으로 나눈다.

- 자원 조건에 따른 `Vallisneria` 클론 통합과 딸 포기 생장을 다룬 연구: [Aquatic Botany — clonal integration in Vallisneria](https://www.sciencedirect.com/science/article/abs/pii/S0304377011001069)
- 빛이 없는 밤에는 수생식물의 산소 발생이 멈추고 햇빛과 함께 다시 시작된다는 고전 관찰: [Nature — Oxygen Generation by the Vallisneria spiralis](https://www.nature.com/articles/014231b0)

## 5. 체리새우 모델과 근거

### 5.1 유지하는 현실 관계

**관찰 근거**는 다음과 같다.

- 체리새우 `Neocaridina davidi`는 표면의 조류·생물막·유기 찌꺼기를 이용할 수 있다.
- 바이오필름 기반 사육계에서 번식부터 어린 개체 성장까지 생활사를 완성할 수 있다.
- 먹이와 몸의 자원은 성장·유지·번식 사이에 배분되며, 번식에는 비용이 따른다.
- 자연 개체군에는 계절적 가입과 크기·성별·수명에 따른 세대교체가 있다.
- 밀도, 수온, 먹이와 수질은 성장·번식·생존에 영향을 줄 수 있다.

### 5.2 현재 구현

체리새우는 개별 개체로 다음 상태를 가진다.

```text
위치·속도·방향
생활 단계·성별·나이·개체 수명
몸·저장 질량에서 계산한 영양 상태·건강
몸 생체량·저장 생체량·번식 저장 생체량
최근 섭식·누적 섭식
번식 휴지기·교미 진행·포란 진행
행동 상태와 먹이 목표
```

새우는 평상시 반경 `64`, 심하게 굶은 상태에서도 반경 `180` 안의 실제 조류 셀만 찾고 접촉 범위에서 먹는다. 그 밖의 군락은 수조 전체를 검색해 알아내지 않으며, 무작위 탐색 이동 중 가까워졌을 때 발견한다. 탐색 후보는 먹이 가치 `0.04` 이상인 눈에 띄는 군락으로 제한하지만, 이미 도달한 군락의 실제 섭식은 Holling II형 밀도 반응을 따라 미량까지 연속적으로 느려진다. 여러 마리가 같은 셀을 먹으면 요청량을 합산한 뒤 남은 조류량을 넘지 않도록 비례 배분한다. 먹힌 조류는 화면과 시뮬레이션 생체량에서 실제로 감소한다.

섭취 질량은 현재 활성 물질 순환에서 다음처럼 나뉜다.

| 경로 | 현재 비율 | 의미 |
|---|---:|---|
| 몸·번식 저장 | 30% | 성장, 유지 후 남은 저장량, 새끼 질량 |
| 고형 배설물 | 42% | 이후 용출되어 분해균 먹이가 됨 |
| 호흡·용존 배설 | 28% | 산소 소비, 무기탄소·암모니아성 노폐물 반환 |

개체는 먹이 부족, 압축 수명 도달, 저산소, 독성 노폐물 누적 또는 극단적인 수온 스트레스에 의해 죽을 수 있다. 사망 시 남은 몸·일반 저장량·번식 저장량만 고형 유기물로 돌아가므로 이미 호흡·배설한 먹이가 사체에서 중복 생성되지 않는다.

영양 상태는 별도의 허기 탱크가 아니다. 먹이의 동화분은 `저장 생체량`에 들어가고 유지 대사와 성장이 이 저장분을 사용한다. 화면의 0~1 영양 상태는 일반 저장 충족도를 72%, 몸 구조의 건전도를 28%로 합성한 파생값이다. 몸 자체를 평상시 먹이 저장고처럼 과대평가하지 않되, 일반 저장량이 고갈되면 생존 가능한 최소 구조량까지 몸을 대사할 수 있다. 먹이가 끊기면 성장과 번식이 먼저 멈추고, 저장량과 가용 구조 질량을 모두 대사한 뒤에만 아사한다.

성체 암컷은 휴지기가 끝난 뒤 일반 저장량 중 최소 체력 비축분 `0.16`을 넘는 부분을 별도의 **번식 저장 생체량**으로 조금씩 옮긴다. 이 저장고가 어린 개체 2마리 질량인 `0.32` 이상 모이고, 반경 `140` 안에서 조건을 만족한 수컷과 실제로 3초간 가까이 있을 때 포란을 시작한다. 포란 질량은 유지 대사와 분리해 보존하며, 75초 뒤 실제 새끼 몸 질량으로 이전한다. 수조 전체에서 먹이나 짝을 검색하는 전역 판정은 사용하지 않는다. 이 구조는 섭취물을 reserve·structure·reproduction buffer로 나누는 단순화된 동적 에너지 예산(DEB) 원칙을 따른다.

수온은 한 개의 만능 계수가 아니라 세 경로로 나뉜다.

```text
대사·산소 요구량 = 24°C 기준량 × 1.07^(현재 수온 - 24)
성장·번식 진행 = 먹이 조건을 만족한 시간 × 종별 번식 수온 곡선
수온 건강 피해 = 1 - 종별 생존 적합도
```

대사 보정은 극단값에서 수치 폭주가 나지 않도록 `0.55~1.65`로 제한한다. 번식 곡선은 `20°C:0.78, 24°C:1.0, 28°C:1.08, 32°C:0.28, 33°C:0`의 핵심점을 사용한다. 이는 20·24·28°C에서 높은 생존을 보이되 낮은 온도에서 대사·번식이 느려지고, 33°C에서 산란이 억제되었다가 28°C 복귀 뒤 회복된 연구 관계를 게임 시간에 맞게 압축한 것이다. 성비는 수온으로 바꾸지 않는다.

플레이와 정보 카드가 참조하는 주요 현재값은 다음과 같다.

| 항목 | 현재값 | 해석 |
|---|---:|---|
| 개체별 압축 수명 | 900~1350초 | 실제 개월 단위 생활사를 실험 시간으로 압축 |
| 지급 성체의 시작 나이 | 180~300초 | 지급 순서를 난수 시드로 쓰되 범위를 고정해 후발 개체의 즉사를 방지 |
| 성숙까지 걸리는 시간 | 180초 | 어린 개체가 즉시 번식하지 않게 함 |
| 먹은 조류의 동화율 | 30% | 실제 저장량으로 들어가 유지·성장·번식에 사용 |
| 초당 최대 섭식량 | 0.375 | 한 개체가 한 셀을 순간 소거하지 않게 제한 |
| 번식/수컷 참여/포란 에너지 | 0.34 / 0.34 / 0.30 | 먹이 없는 번식을 방지하는 자원 문턱 |
| 저산소 스트레스 시작 | 산소 30 | 실제 mg/L가 아닌 게임 농도 |
| 독성 스트레스 시작/최대 | 노폐물 6 / 24 | 문턱 뒤 연속적으로 피해 증가 |
| 초당 최대 산소/독성 피해 | 0.025 / 0.032 | 순간 접촉이 아닌 누적 건강 피해 |
| 건강한 물의 초당 회복 | 0.004 | 환경 회복 뒤 생존 가능 |
| 대사 수온 기준/계수 | 24°C / 1.07 | 1°C당 theta 보정, `0.55~1.65` 제한 |
| 번식 정지 수온 | 33°C | 대사와 생존을 즉시 0으로 만들지 않고 번식 진행만 정지 |

### 5.3 게임 보정과 한계

- 실제 10~15개월 규모의 생활사를 `900~1350` 시뮬레이션초로 압축한다.
- 실제 산란 수를 그대로 화면 개체 수로 만들지 않고 한 번의 포란을 2~3개의 대표 어린 개체로 압축한다.
- 각 압축 brood에는 암수가 모두 포함되도록 생성 순서를 보정한다. 이는 실제 성 결정 규칙이 아니라 작은 표시 개체군의 우연한 단절을 줄이는 게임 장치다.
- 현재 먹이는 두 조류뿐이다. 현실의 생물막, 세균, 원생생물, 찌꺼기를 모두 별도 먹이로 구분하지 않는다.
- 저산소 시작값 `30`, 독성 시작값 `6`은 mg/L가 아닌 게임 농도다. 실제 사육 안전값으로 인용하면 안 된다.
- 20~28°C의 상대 관계는 연구와 맞추되 곡선 중간값과 피해 속도는 압축 수명에 맞춘 게임 보정이다.
- 33°C에서 번식이 억제된다는 근거를 사망 문턱으로 오해하지 않는다. 직접 수온 피해는 더 넓은 생존 곡선에서 누적한다.
- 온도 의존 성비는 근거의 재현성과 작은 개체군 안정성을 고려해 현재 모델에 넣지 않았다.

## 6. 분해균·질산화군집 모델과 근거

### 6.1 기능군으로 나누는 이유

현재 모델은 미생물을 실제 종 이름으로 고정하지 않고 두 기능군으로 나눈다.

1. **종속영양 분해자 필름**: 유기물과 산소를 사용하고, 일부를 새 필름으로 고정하며 나머지를 무기탄소와 암모니아성 질소로 돌린다.
2. **통합 질산화 필름**: 암모니아성 독성 노폐물과 산소를 사용하고, 일부를 새 필름으로 고정하며 나머지를 영양염으로 바꾼다.

실제 수조의 질산화는 암모니아 산화 세균·고세균, 아질산 산화 미생물과 완전 암모니아 산화 `comammox` 등 여러 구성원이 담당할 수 있다. 따라서 도감의 `질산화균`은 특정 한 종이 아니라 게임의 통합 질산화군집이라는 점을 명시한다.

### 6.2 현재 구현

두 기능군은 먹이와 산소가 모두 필요하다는 이중 Monod형 제한을 사용한다.

```text
활동도
= 먹이 / (먹이 + 먹이 반포화값)
 × 산소 / (산소 + 산소 반포화값)

처리량
= 필름량 × 최대 처리율 × 활동도 × 수온 보정 × dt

성장
= 실제 처리량 × 생체 전환율 × 빈 표면

사멸
= [유지 사멸 + 기아 사멸 × (1 - 실현 활동도)] × 수온 보정

수온 보정
= 1.08^(현재 수온 - 24°C)
```

현재 주요 상대 관계는 다음과 같다.

- 분해자는 유기물 펄스에 빠르게 증가하고 먹이가 줄면 빠르게 감소한다.
- 질산화군집은 더 느리게 성장하지만 낮은 부하에서 더 오래 유지된다.
- 두 군집 모두 산소가 부족하면 처리와 성장이 줄어든다.
- 같은 표면 공간을 공유하고 과밀하면 일부가 떨어져 고형 유기물이 된다.
- 인접 표면 확산과 작은 수중 방출·재정착을 통해 떨어진 표면으로도 천천히 전파된다.

현재 반응 상수는 다음과 같다. 모두 게임 농도와 시뮬레이션초에 맞춘 상대값이다.

| 기능군 | 먹이 반포화 | 산소 반포화 | 최대 처리율 | 생체 전환율 | 유지/기아 사멸률 | 산소 수지 | 표면 전파율 |
|---|---:|---:|---:|---:|---:|---|---:|
| 분해자 | 4 | 14 | 0.11 | 0.42 | 0.0022 / 0.013 | 실제 무기화된 유기탄소에만 부과 | 0.025 |
| 통합 질산화군집 | 5 | 24 | 0.027 | 0.11 | 0.0012 / 0.0035 | 실제 질산염 생성과 균체 탄소 고정을 함께 계산 | 0.012 |

두 기능군의 현재 처리율은 24°C에서 기존 미션 균형과 같게 정규화했다. EPA WASP가 질산화와 용존 유기질소 무기화에 사용하는 `θ=1.08`의 형태와 상대 온도 민감도를 적용하되, 게임의 초 단위 처리율 자체를 실제 일 단위 상수로 주장하지 않는다. 먹이·산소가 같아도 차가운 표면에서는 증식·사멸·전환이 함께 느리고 따뜻한 표면에서는 빨라진다.

분해자와 질산화군집의 수중 이탈률은 각각 `0.0007`, `0.00035`, 부유 상태 사멸률은 각각 `0.025`, `0.008`이다. 이 값은 유기물 펄스 뒤 분해자가 먼저 증가·감소하고 질산화군집이 더 늦게 반응하는 지연 되먹임을 만들도록 조정했다.

### 6.3 게임 보정과 한계

- 반포화값·처리율·수율은 실제 mg/L 또는 일 단위 상수가 아니라 압축 게임 농도에 맞춘 값이다.
- 암모니아→아질산→질산의 두 단계를 `toxicWaste → nutrients` 한 단계로 묶는다.
- pH, 알칼리도, 규산염, 인산염, 혐기성 분해와 탈질은 계산하지 않는다.
- 필름 내부의 산소·기질 침투 깊이는 풀지 않고 주변 반경 수질과 하나의 산소 제한항으로 압축한다.
- 미생물 종 조성과 천이는 표시하지 않고 기능군 생체량만 보여준다.
- `θ` 보정은 배포본의 정상 수온 범위를 위한 반응속도 근사다. 극단적인 고온에서 실제 군집이 붕괴하는 종 조성 변화까지 풀지 않는다.

## 7. 산소·탄소·질소 물질 장부

### 7.1 현재 구현 원칙

현재 미션 5·6과 실험실은 빛 에너지만 외부에서 받고 물질은 닫힌 단순계로 다룬다.

```text
무기탄소 + 무기질소
→ 조류 생체량 + 산소
→ 새우 몸·배설물·호흡 산물
→ 고형·용존 유기물
→ 분해균 성장 또는 호기성 무기화
→ 암모니아성 노폐물
→ 질산화군집 성장 또는 질산화
→ 영양염
→ 다시 조류
```

모든 생물과 유기물은 게임용 고정 C:N 조성을 공유한다. 이 비율은 실제 모든 생물의 원소 조성이 같다는 주장이 아니라, 섭식·배설·사망 과정에서 질량이 이중 생성되거나 사라지는지를 감사하기 위한 압축 장부다.

물속 무기탄소와 공기층 CO₂, 용존산소와 공기층 O₂는 각각 유한한 저장고 사이에서만 교환한다. 물속 무기탄소는 `36×20` 공간장이므로 조류·질산화균 주변의 국소 탄소 소비와 공통 물 흐름에 따른 이동이 나타난다.

현재 장부는 생체량 `1.0`에 질소 `0.08`, 탄소 `0.32`를 대응시킨다. 고정·무기화되는 유기탄소 `1.0`당 산소 등가량 `1.12`를 양방향에 똑같이 사용한다. `1.12`는 게임 표시 단위의 환산척도이며 더 이상 장기 실행을 맞추기 위해 생산 쪽에만 더한 여유 계수가 아니다.

질산화의 산소 등가량은 독립 보정값으로 맞추지 않는다. 탄소 산화의 실제 질량비 `32/12 g O₂/g C`와 암모니아성 질소 산화의 `64/14 g O₂/g N` 비율로부터 다음처럼 유도한다.

```text
qC = 1.12
qN = qC × (64/14) ÷ (32/12) = 1.92
```

전체 반응은 상태 벡터 `x`, 화학량론 행렬 `S`, 반응속도 `v`에 대해 다음 형태로 해석한다.

```text
dx/dt = S · v(x)
```

반응속도는 광량·수온·먹이·산소·표면 공간에 따라 달라지지만, 각 반응 열은 탄소·질소와 아래 산소 등가 불변량을 보존해야 한다.

```text
E = (물속 산소 + 공기층 산소)
    - qC × 전체 유기탄소
    + qN × 질산염성 영양질소
```

닫힌 수조 내부 반응과 물–공기층 재분배만 있을 때 `E`의 변화는 수치 허용오차 안에서 0이어야 한다.

#### 생산자

생산자가 무기탄소 `Cfixed`, 질산염성 질소 `Nnitrate`를 새 생체량으로 고정할 때:

```text
O₂ 생산 = qC × Cfixed + qN × Nnitrate
```

암모니아성 질소는 이미 생물체 질소와 같은 환원 상태로 압축하므로 추가 `qN` 항이 없다. 질산염은 생물체 질소로 환원할 때 1 mol N당 8전자, 즉 2 mol O₂에 대응하는 광합성 전자 흐름이 필요하므로 산소 **등가량**을 더한다. 이는 질산염 흡수 시 별도 산소 기포가 즉시 보인다는 뜻이 아니라, 생략한 전자전달계를 닫는 장부 항이다.

생산자 호흡은 정확히 반대다.

```text
O₂ 소비 = qC × 실제 무기화 탄소
```

산소가 부족하면 수행 가능한 호흡량만 생체량에서 빠진다. 요청했지만 수행하지 못한 호흡량을 생체량에서 먼저 제거하지 않는다.

#### 새우

섭식은 유기탄소의 소유권을 바꾸는 과정이다. 동화된 몫은 몸과 번식 저장량, 배설된 몫은 찌꺼기로 이동하며 이 이동 자체에는 산소를 부과하지 않는다. 실제 호흡으로 유기탄소가 무기탄소가 된 몫에만 `qC`를 적용한다. 산소가 부족해 호흡하지 못한 먹이 몫은 삭제하지 않고 유기 찌꺼기에 남긴다. 사망 역시 남은 몸 질량을 찌꺼기로 옮길 뿐 즉시 산소를 소비하지 않는다.

#### 분해균

유기물 섭취량 `U`, 실제 균 성장량 `G`라면:

```text
실제 무기화량 = U - G
O₂ 소비 = qC × biomassCarbon × (U - G)
```

이전 구현은 `U` 전체에 산소를 부과했다. 균체로 남은 `G`가 나중에 죽어 다시 분해될 때 산소가 또 부과되어 같은 탄소가 경로에 따라 이중 과금되는 문제가 있었다. 현재는 실제 무기탄소가 된 몫에만 부과한다.

#### 질산화군집

처리한 암모니아성 질소 `Nprocessed` 중 새 균체에 남은 질소를 `Nretained`, 생성한 질산염을 `Nnitrate`, 고정한 균체 탄소를 `Cfixed`라 하면:

```text
Nnitrate = Nprocessed - Nretained
O₂ 소비 = qN × Nnitrate - qC × Cfixed
```

균체로 동화된 질소에는 질산화 비용을 부과하지 않으며, 암모니아 산화에서 얻은 환원력이 균체 탄소 고정에 저장된 몫도 차감한다. 계산 결과는 음수가 되지 않도록 제한한다.

#### 동시 반응과 수치 정밀도

균 반응은 한 기능군이 같은 스텝에서 방금 만든 산물을 다음 기능군이 즉시 먹지 않도록 두 단계로 처리한다.

1. 모든 필름이 반응 전 농도장에서 처리 요청량을 계산한다.
2. 기질·산소·무기탄소 인출 장부와 생성물 수용 장부를 분리해 제한한 뒤 한꺼번에 반영한다.

표면 셀 배열 순서를 뒤집어도 같은 결과가 나오는 회귀 시험을 둔다. 보존 농도장은 장시간 반복 반올림을 막기 위해 `Float64Array`를 사용하며, 화면용 스냅샷에서만 일반 숫자 배열로 복사한다.

미션 5의 초기 물속 무기탄소는 `58`, 닫힌 공기층 CO₂ 저장고는 `22`이며 공기층 산소는 시나리오 초기 산소와 같은 값으로 시작한다. 이 값들은 실제 몰수나 mg/L가 아니라 여러 세대 동안 순환을 관찰하기 위한 보정값이다.

### 7.2 현재 수온–산소 기체 교환

실제 물은 온도가 높아질수록 일반적으로 산소 용해도가 낮아진다. 현재 구현은 수면과 맞닿은 첫 행의 평균 수온을 USGS가 채택한 Benson–Krause 담수 산소 용해도 식에 넣는다.

```text
DOsat(T) = exp(
  -139.34411
  + 1.575701×10^5 / K
  - 6.642308×10^7 / K²
  + 1.243800×10^10 / K³
  - 8.621949×10^11 / K⁴
)

상대 물쪽 용량 S(T) = DOsat(T) / DOsat(24°C)
물속 평형 산소 = (물속 산소 + 공기층 산소) × S / (1 + S)
```

`DOsat` 값만 mg/L이며 게임의 용존산소 `0~100`을 mg/L라고 부르지 않는다. 24°C에서 기존 물/공기층 1:1 분배를 보존하고, 차가워지면 같은 총산소 중 물쪽 몫이 커지며 따뜻해지면 공기층으로 이동한다. 이동량은 수면 셀에만 더하거나 빼고 반대 저장고에 같은 양을 기록하므로 온도가 산소를 생성·삭제하지 않는다.

물속 무기탄소는 용존 CO₂만이 아니라 pH를 생략한 완충 무기탄소 저장고다. 따라서 탄산 평형을 풀지 않은 상태에서 전체 무기탄소에 CO₂ Henry 상수를 직접 적용하지 않고 기존의 약한 물–공기층 교환을 유지한다.

또한 광합성은 산소를 만들고 호흡·유기물 분해·질산화는 산소를 소비한다. 물 흐름과 난류 혼합은 이렇게 생성·소비된 산소를 다른 위치로 운반한다. 현재 공통 유동장은 생물 반응의 크기를 대체하지 않고 반응 산물의 이동 경로를 제공한다.

| [MOM6 convective adjustment](https://mom6.readthedocs.io/en/dev-gfdl/api/generated/modules/mom_bulk_mixed_layer.html) | 해상하지 않은 불안정 수직층을 혼합하는 대안과 검증 참고 | 공통 속도장을 채택하면 주 대류 계산으로 중복 사용하지 않음 |
| [AED2 공식 모델 라이브러리](https://aed.see.uwa.edu.au/research/models/AED/download.html) | 물리 host가 수송·수온을 공급하고 생태 모듈이 국소 조건으로 반응하는 계층 분리 | 대형 호수용 전체 상태변수와 상세 매개변수는 제외 |
| [US EPA WASP 문서](https://www.epa.gov/hydrowq/wasp-model-documentation) | 물리 수송과 수질 반응 모듈 분리, 반응별 theta 온도 보정 | 실제 하천·만의 일 단위 보정값을 게임 초 단위로 복사하지 않음 |

## 9. 게임·관찰 도구 참고 자료

이 자료들은 생물학 상수의 출처가 아니라 **학습 순서와 관찰 도구 설계**의 참고다.

- [Cell Lab 공식 사이트](https://cell-lab.net/): 도전 과제와 자유 실험을 함께 제공하고, 제한된 조건에서 시행착오로 시스템을 이해하게 하는 구조를 참고했다. AquaCycle은 종의 유전 형질 편집 대신 종 선택·공간 배치·환경 조절을 사용한다.
- [Tyto Ecology](https://store.steampowered.com/app/453750/Tyto_Ecology/): 생산자·소비자·분해자를 포함한 자립 생태계, 도전 과제, 데이터 지도를 함께 제공하는 구성을 참고했다. 특정 풀이를 지시하지 않고 결과를 관찰하게 하는 방향을 유지한다.
- [JavaLab Ecosystem](https://javalab.org/en/ecosystem_en/): 개체의 활력·영양 상태, 실제 섭식과 번식이 개체군 증감으로 이어지는 단순한 소비자–자원 예제로 참고했다. AquaCycle은 별도의 숨은 개체 수 상한보다 실제 먹이 생산과 소비를 우선한다.
- [The Sapling](https://thesaplinggame.com/): 복잡한 생태·진화 요소를 단계적으로 공개하고 이후 자유 실험으로 확장하는 구조를 참고했다.

## 10. 관찰 UI에 공개해야 할 값

모델이 물리적으로 그럴듯해도 플레이어가 원인을 볼 수 없으면 게임 규칙으로 기능하지 못한다. 현재 관찰 지도에는 다음 범위가 연결되어 있다.

- 위치별 수온과 수조 평균·최저·최고 수온
- 선택 위치의 유속 방향과 상대 속도
- 선택한 수질의 농도, 최근 증가·감소 및 주요 원인
- 수조 전체 질소·탄소·산소 등가 장부와 기준 대비 상대 오차
- 선택한 새우의 현재 수온, 대사·번식 수온 보정, 생존 적합도
- 선택 표면의 두 균 기능군 수온 보정과 수온이 반영된 예상 순성장률
- 수면 수온, 1기압 담수 산소 포화도, 닫힌 계의 현재 물쪽 산소 평형 목표

광원·경계의 열 출입 분해, 구조물 자체 온도와 열 장부 오차의 UI는 후속 범위다.

벡터 화살표를 화면 전체에 항상 표시하지 않는다. 기본 화면은 현재 두들 스타일을 유지하고, 관찰 지도에서 `수온`, `물 흐름`, `산소` 등을 필요할 때 겹쳐 본다. 숫자 탐침은 현재 위치의 값과 짧은 시간 추세를 제공한다.

TypeScript 정적 검사와 Electron Forge macOS arm64 production 패키징도 완료했다. 현재 실행 명령·수치와 검증 계약은 `docs/RELEASE_VERIFICATION.md`, 세부 수온 기록은 `docs/TEMPERATURE_COUPLING_IMPLEMENTATION.md`, 화학량론 구현·수식·장주기 수치는 `docs/STOICHIOMETRIC_LEDGER_IMPLEMENTATION.md`에 정리했다.

단계마다 이전 단계와 같은 초기 상태를 저장해 총량, 공간 분포, 미션 결과와 Worker 시간을 비교한다. 열과 유동을 한 번에 생태계 전체에 연결하지 않는다.

## 12. 아직 결정하지 않은 것

- 실제 부피·기압·염분을 도입해 게임 산소 `0~100`을 mg/L로 교정할지
- 규산염을 규조류 전용 제한 자원으로 추가할지
- pH·알칼리도와 암모니아/암모늄 비율을 언제 분리할지
- 필터·에어레이션·히터·냉각기 같은 능동 장치를 공통 유동장에 추가할 시점
- 2D 구조물의 앞뒤 우회를 나타내는 기본 투과도와 관찰 표현
