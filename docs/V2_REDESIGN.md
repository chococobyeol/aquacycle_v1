# AquaCycle V2 재설계 명세

> 문서 상태: 구현 기준·현재 플레이어블 반영  
> 작성일: 2026-07-13 · 최근 갱신: 2026-07-20  
> 적용 대상: 네 개 도전 미션과 체리새우를 포함한 실험실  
> 이전 구현: V1에서 확인한 문제를 재설계한 현재 프로토타입을 기준으로 한다.

이 문서는 실제 실행본에서 발견한 문제와 V2에서 지켜야 할 규칙, 이후 추가된 생산자 미션과 첫 소비자 시스템을 하나의 구현 기준으로 기록한다. 기존의 장기 생태계 기획은 [핵심 플레이 기획](./CORE_GAMEPLAY.md), 기술 선택의 근거는 [초기 MVP 기술 구조](./MVP_TECHNICAL_ARCHITECTURE.md)에 남겨 둔다.

V2의 현재 목적은 **구조물을 놓고, 빛을 측정하고, 적합한 표면에 조류를 접종해 실제로 성장·사멸하는 과정에서 출발해, 체리새우가 그 조류를 실제로 먹고 에너지와 개체군이 변하는 첫 먹이망까지 납득할 수 있게 만드는 것**이다.

---

## 1. 이번 재설계에서 확정한 것

1. 게임은 2D를 유지한다.
2. 위젯은 이번 구현 범위에서 제외하고 일반 게임 창의 진행 경험부터 완성한다.
3. 도전 모드와 실험실을 모두 제공한다.
4. 도전 모드는 시뮬레이션 시작 후 플레이어의 배치 편집을 잠근다.
5. 실험실은 일시정지 중 구조물과 접종체를 편집할 수 있다.
6. 돌은 플레이어가 이동·회전해 놓을 수 있고, 놓은 뒤에는 중력·충돌·마찰·토크에 따라 자연스럽게 안착한다.
7. 돌의 보이는 앞면 전체를 2D 생태 표면으로 취급한다. 조류가 외곽선 한 줄에만 자라지 않는다.
8. 초기 범위에서 조류가 자랄 수 있는 곳은 돌의 앞면과 바닥재 표면이다. 수조 유리는 제외한다.
9. 구조물의 크기 조절 기능은 제공하지 않는다. 크기·형태·재질이 다른 완성형 구조물을 제공한다.
10. 구조물과 생물은 목록에서 꺼내 배치하며, 이동과 새 배치를 서로 다른 상태로 구분한다.
11. 광량 탐침은 우선 숫자를 표시한다. 종을 선택했을 때만 그 종의 실제 성장 계산과 같은 기준으로 성장 경향을 덧붙인다.
12. 조류는 거대한 개체 그림이 아니라 군락·막·미세한 털의 형태로 표현한다.
13. 종 설명과 해금 상태를 확인할 수 있는 도감/상세 패널을 제공한다.
14. 한국어 UI의 주 글꼴은 `오뮤 다예쁨체` 보통 굵기로 사용한다. 작은 수치와 학명은 시스템 산세리프를 병용한다.
15. 체리새우는 조류 군락과 별도의 개별 개체로 계산하며, 실제 섭식·에너지·성장·번식·사망을 가진다.
16. 체리새우 이동은 Matter.js 강체가 아니라 연속 좌표 기반 운동학적 조향으로 계산한다.
17. 산소 생산·소비, 배설과 사망 질량은 공간별 ledger에 기록하되 현재는 생존·성장·미션과 UI에 영향을 주지 않는다.

정확한 미션 제한시간, 목표 점유율, 성장률은 설계 미결정 사항이 아니라 **플레이 테스트로 조정할 밸런스 값**으로 분리한다.

## 2. V1에서 확인한 문제와 원인

### 2.1 구조물 모양이 서로 맞지 않음

V1은 다음 세 가지를 서로 다른 근사치로 만들었다.

- 화면에 보이는 돌 스프라이트
- Matter.js가 사용하는 충돌 도형
- 조류가 붙는 표면 경로

그 결과 둥근 돌도 사각형처럼 충돌하고, 조류는 돌 앞면이 아니라 임의의 윗선에 붙었다. V2에서는 세 데이터가 반드시 **같은 오브젝트 로컬 좌표계**를 사용한다.

### 2.2 회전하면 돌이 늘어남

수조를 창 크기에 맞출 때 X축과 Y축을 서로 다른 비율로 확대했고, 스프라이트의 원본 비율과 무관하게 너비·높이를 강제로 지정했다. 비등방 스케일 상태에서 회전하면 같은 돌의 가로세로 비율이 달라 보인다.

V2는 수조 전체를 하나의 동일 비율로 확대한다. 플레이 화면은 일반 게임 카메라처럼 창을 전부 채우고, 창과 수조의 화면비가 다르면 한 축을 조금 잘라낸다. 모든 입력 좌표도 이 변환의 역행렬로 수조 좌표에 바꾼다.

### 2.3 회전 조작이 지나치게 느림

V1 회전 버튼은 한 번에 약 6.9도만 회전해 반복 클릭이 필요했다. V2는 버튼 한 번에 15도, 마우스 휠은 약 5도 단위로 회전한다. 버튼을 누르고 있으면 연속 회전한다.

### 2.4 배치와 이동의 상태가 불명확함

V1은 드래그 중에만 물체가 따라오고, 접종체 수량을 모두 쓴 뒤 다시 배치하면 기존 접종체를 조용히 다른 곳으로 옮겼다. 사용자는 새 개체를 놓는지 기존 개체를 옮기는지 알 수 없었다.

V2는 `목록에서 새로 꺼냄`, `기존 오브젝트 선택`, `들고 있음`, `배치됨`을 명시적인 상태로 나눈다. 수량이 0이면 새 배치 버튼이 비활성화되며 자동 재배치하지 않는다.

### 2.5 광량 퍼즐이 공간적으로 충분히 달라지지 않음

V1 광량은 수심과 가림 여부의 영향이 주였고, 고정 조명에서 좌우 거리와 조사 각도의 영향이 작았다. 따라서 돌을 어느 위치에 놓아도 결과가 비슷해질 수 있었다.

V2는 광원과의 거리, 조사각, 수심 감쇠, 구조물 가림과 약한 산란광을 함께 계산한다.

### 2.6 성장·사멸과 탐침 판정이 서로 다름

V1의 `부족/적정/과다`는 모든 종에 공통인 고정 구간이었지만 실제 성장 계산은 종별 곡선을 사용했다. 일부 조건에서는 UI가 과다라고 표시해도 조류가 계속 성장할 수 있었다. 이 표시는 제거하고 시뮬레이션, 탐침, 도감 그래프가 하나의 종별 반응 함수를 공유한다.

### 2.7 조류의 화면 크기가 생태적 단위와 맞지 않음

붓뚜껑말과 규조류를 큰 촉수나 타원 개체처럼 그려 실제 조류보다 말미잘에 가까워 보였다. V2에서 현미경적 개체는 실물 크기로 직접 그리지 않는다. 수조 화면에는 사람이 육안으로 보는 군락의 색·밀도·막을 표현하고, 개별 세포 형태는 도감의 확대 보기에서만 보여준다.

### 2.8 돌 하나가 떨어져도 끊김

V1은 움직이는 동안 광량 레이 질의와 전체 광량 격자 그리기, 구조물 표시 객체 재생성을 자주 반복했다. 물리와 무관한 고비용 작업이 낙하 프레임에 함께 실행된 것이 주원인이다.

V2는 움직이는 동안 물리와 저비용 미리보기만 갱신하고, 정밀 광량장은 구조물이 안착한 뒤 한 번 계산한다. 화면 객체는 없애고 다시 만드는 대신 유지한 채 변경된 값만 갱신한다.

## 3. V2 플레이 범위

### 3.1 포함

- 일반 창에서의 네 개 도전 미션
- 네 개 미션에서 공개된 구조물, 두 조류와 체리새우를 조절할 수 있는 실험실
- 구조물 목록, 생물/접종체 목록과 측정 장비 목록
- 돌의 선택, 이동, 수동 회전, 중력 낙하, 충돌, 토크와 자동 안착
- 고정 조명, 공간 광량장, 구조물 그림자와 광량 탐침
- 돌 앞면·바닥재 위의 조류 접종, 성장, 확산, 경쟁과 사멸
- 붓뚜껑말속과 `Nitzschia palea`의 상세 정보 및 해금 표시
- 체리새우 개별 배치, 운동학적 탐색·이동, 실제 조류 섭식, 에너지, 어린 개체 성장, 성체 암수 번식과 사망
- 잠재 산소 생산·소비, 배설물과 사망 질량의 관찰 전용 공간 ledger
- 미션 목표, 진행률, 성공·실패와 재시작

### 3.2 제외

- 위젯 창
- 물고기, 달팽이, 박테리아와 체리새우 이후의 먹이망
- 영양염, 용존산소, 배설물·사체 오염 농도, 물 흐름과 박테리아 분해의 실제 동적 계산
- 산소·오염·사체가 생물 건강이나 미션 판정에 미치는 영향
- 구조물 크기 조절
- 조류의 유전·형질 편집
- 정교한 유체역학과 3D 광학
- 아이콘과 메뉴의 최종 아트 품질

수온은 잘 섞인 수조의 전역값으로 계산되어 조류 성장에 사용한다. 영양염·산소·오염과 분해는 아직 활성 환경 변수가 아니다. 미션 1~3의 실패 원인은 광량과 조류 성장 환경으로, 미션 4의 실패 원인은 실제 먹이 섭취에 따른 새우 에너지 고갈로 제한한다.

## 4. 화면과 도구 구성

### 4.1 기본 화면

```text
┌──────────────────────────────────────────────────────────┐
│ [메뉴][보유] 미션 이름       실행 시간    [퀘스트][관찰] │
│                                                          │
│                    크게 표시되는 수조                    │
│                                                          │
│                                      [이동][-][100%][+]   │
│ 현재 상태                         시작/일시정지 · 시간배속 │
└──────────────────────────────────────────────────────────┘
```

보유 목록, 퀘스트, 관찰 기록과 설명은 기본적으로 닫혀 있다. 상단 아이콘으로 필요한 패널만 독립적으로 열며, 열린 패널은 기존 종이·메모지 스타일의 고정된 포스트잇으로 수조 위에 겹쳐 표시한다. 패널을 열어도 수조 자체를 재배치하거나 비등방으로 줄이지 않는다. 창이 좁아져도 수조의 논리 좌표와 비율은 변하지 않으며, 열린 패널 내부만 스크롤한다.

### 4.2 보유 목록

목록은 세 범주로 나눈다.

- `구조물`: 돌의 썸네일, 이름, 크기, 재질과 남은 수량
- `생물`: 접종체 썸네일, 한국어 이름, 학명, 남은 수량과 해금 상태
- `측정 장비`: 광량 탐침과 광량 오버레이

구조물은 카드의 그림만 보고도 실제 모양을 예상할 수 있어야 한다. V2는 최소한 납작한 돌, 둥근 돌, 세로로 긴 돌을 제공하며 세 종류 모두 사각형이 아닌 고유 외곽선과 충돌 형태를 가진다.

### 4.3 선택 정보와 도감

종 카드나 수조의 군락을 선택하면 다음 정보를 표시한다.

- 한국어 이름과 학명
- 실제 개체의 대략적인 크기
- 수조에서 보이는 군락 형태
- 현미경 확대 형태
- 부착 가능한 표면
- 상대 광량에 따른 순성장 곡선
- 현재 선택 위치의 예상 변화: `성장`, `거의 유지`, `감소`
- 다른 종과의 경쟁상 특성
- 해금 조건

아직 잠긴 종은 실루엣, 역할과 해금 조건만 표시한다.

## 5. 배치와 입력 규칙

### 5.1 한 번 클릭해 들고, 다시 클릭해 놓기

1. 목록의 구조물 카드를 클릭하면 새 구조물이 커서에 붙는다.
2. 기존 구조물을 클릭하면 그 구조물을 집는다.
3. 마우스 버튼을 누르고 있지 않아도 구조물이 커서를 따라온다.
4. 유효한 위치에서 다시 클릭하면 놓는다.
5. 놓은 직후 동적 물리가 적용되어 떨어지고 회전하며 안착한다.
6. 유효하지 않은 위치에서는 반투명한 붉은 외곽선을 보이고, 클릭해도 계속 들고 있는다.
7. `Esc` 또는 우클릭은 취소한다. 기존 구조물은 집기 전 위치로 돌아가고, 새 구조물은 목록 수량으로 돌아간다.

### 5.2 회전

- 들고 있는 구조물만 플레이어가 직접 회전할 수 있다.
- 휠: 약 5도씩 회전
- 회전 버튼 클릭: 15도씩 회전
- 회전 버튼 길게 누르기: 일정 속도로 연속 회전
- 놓은 다음에는 플레이어가 지정한 각도를 강제로 유지하지 않는다.
- 중력과 접점의 위치가 만드는 토크에 따라 다시 회전할 수 있다.

### 5.3 접종체

- 생물 목록의 접종체 카드를 클릭하면 접종 마커가 커서에 붙는다.
- 접종 가능한 생태 표면 위에서만 놓을 수 있다.
- 수량을 모두 사용하면 새 접종 버튼은 비활성화된다.
- 준비 단계에는 `선택/이동` 도구로 이미 놓은 접종체를 옮길 수 있다.
- `회수`를 선택하면 접종체가 목록 수량으로 돌아간다.
- 도전 모드에서 시뮬레이션을 시작한 뒤에는 접종체나 성장한 군락을 움직일 수 없다.
- 실험실에서는 일시정지했을 때만 접종체를 이동·회수할 수 있다. 이미 성장한 군락을 마음대로 옮기는 기능은 제공하지 않는다.

### 5.4 모드별 편집 권한

| 상태 | 도전 모드 | 실험실 |
|---|---|---|
| 시작 전 | 구조물·접종체 편집 가능 | 구조물·접종체 편집 가능 |
| 실행 중 | 편집 불가 | 편집 불가 |
| 일시정지 | 편집 불가 | 구조물·접종체 편집 가능 |
| 재시작 | 초기 지급 상태로 복원 | 현재 설정 또는 초기화 선택 |

실험실에서 구조물을 움직이면 그 구조물 로컬 좌표에 저장된 군락도 함께 움직인다. 다만 이를 성장 최적화 꼼수로 쓰지 않도록 실험실 결과는 도전 미션 기록에 반영하지 않는다.

## 6. 구조물의 공통 기하 데이터

각 구조물 에셋은 하나의 로컬 좌표계에서 다음 데이터를 가진다.

```ts
interface StructureGeometry {
  id: string;
  nativeSize: { width: number; height: number };
  visualSilhouette: Polygon | AlphaMask;
  collisionPolygon: Polygon;
  ecologyFaceMask: Mask;
  lightOccluder: Polygon;
  inoculationMask?: Mask;
  pivot: { x: number; y: number };
}
```

### 6.1 시각 외곽선

- 원본 두들 이미지는 고유 종횡비를 유지한다.
- 코드에서 너비와 높이를 따로 강제하지 않는다.
- 선택 외곽선은 사각 바운딩 박스가 아니라 실제 실루엣을 따른다.

### 6.2 충돌 도형

- 돌 하나당 대략 8~16개 꼭짓점의 볼록 다각형 또는 소수의 볼록 조각으로 만든다.
- 충돌 도형은 그림의 투명 영역을 침범하지 않도록 외곽선보다 조금 안쪽에 둔다.
- 복잡한 화상 픽셀 충돌은 사용하지 않는다.
- 개발용 디버그 보기에서 스프라이트 위에 충돌선을 겹쳐 확인할 수 있어야 한다.

### 6.3 생태 앞면 마스크

- 보이는 돌의 앞면 전체가 조류 성장 후보 영역이다.
- 외곽선에서 2~4 논리 픽셀 정도 안쪽을 기본 성장 마스크로 사용한다.
- 갈라진 틈이나 조류가 붙으면 안 되는 재질 영역은 별도 마스크로 뺄 수 있다.
- 마스크는 약 6 논리 픽셀 간격의 생태 셀로 래스터화한다. 아주 작은 돌에도 최소 셀 수를 보장하도록 에셋별로 조정할 수 있다.
- 조류는 경계선뿐 아니라 마스크 내부의 밝은 면, 어두운 면, 가려진 부분에 서로 다른 밀도로 자란다.

충돌 도형과 생태 마스크는 목적상 완전히 같을 필요는 없지만, 같은 로컬 좌표를 사용하고 눈으로 봤을 때 어긋나지 않아야 한다.

### 6.4 2D에서 앞면 성장을 해석하는 방법

돌의 그림은 3D 물체를 옆에서 본 두들 그림이지만, 생태 계산에서는 그 그림의 채워진 면을 펼쳐진 서식 표면으로 본다. 이는 실제 표면적을 복원하는 3D 계산이 아니라 다음 플레이 규칙을 위한 의도적인 추상화다.

- 화면에서 보이는 돌 앞면에 조류가 얼룩이나 막처럼 퍼진다.
- 구조물의 회전과 이동에 따라 마스크와 군락도 함께 움직인다.
- 다른 구조물이 앞을 가리면 가려진 부분의 광량이 낮아진다.
- 돌의 보이지 않는 뒤쪽은 첫 두 미션에서 계산하지 않는다.

이 규칙으로 3D 조작 없이도 돌의 위치·회전·겹침이 성장 공간을 바꾸게 한다.

## 7. 물리 시뮬레이션

### 7.1 물리는 배치 순간에만 존재하지 않음

물리 상태는 오브젝트마다 유지한다.

```text
held    플레이어가 들고 있음, 물리 제외
dynamic 중력·충돌·토크가 계산됨
sleeping 안정되어 계산 비용을 거의 쓰지 않음
reactivated 지지대 이동·충돌·물성 변화로 다시 dynamic
anchored 바닥재처럼 설계상 고정됨
```

첫 두 미션의 돌은 놓은 뒤 `dynamic`이 되고, 충분히 안정되면 `sleeping`이 된다. 완전히 굳혀 버리는 것이 아니므로 지지하던 돌이 움직이거나 향후 부력·재질 변화 이벤트가 생기면 다시 활성화할 수 있다.

### 7.2 안착 판정

- 선속도와 각속도가 작은 상태가 일정 시간 지속되면 잠든다.
- 좁은 접점이라도 실제 힘의 균형이 맞으면 허용한다.
- 돌 쌓기의 개수나 높이에 별도 규칙 제한을 두지 않는다.
- 수조 밖으로 나가거나 서로 깊이 겹치는 배치는 놓기 전에 막는다.
- 잠든 뒤 광량장과 성장 마스크의 월드 위치를 정밀 재계산한다.

정확한 속도 임계값과 대기 시간은 테스트로 조정하되, 화면상 흔들리는 중에 성급하게 고정되거나 영원히 미세 진동하지 않아야 한다.

### 7.3 미래 확장

식물의 기포 축적, 부력 변화, 생물 충돌 같은 후속 기능은 오브젝트의 질량·밀도·부력·활성 상태를 바꾸고 잠든 물체를 깨우는 사건으로 구현할 수 있다. 따라서 V2에서부터 물리를 `배치가 끝났으니 삭제`하는 구조는 사용하지 않는다.

## 8. 좌표와 화면 스케일

- 시뮬레이션 수조는 창 크기와 독립된 고정 논리 좌표를 사용한다.
- 논리 수조의 기준은 현재처럼 `1200 × 720`을 유지할 수 있다.
- 카메라 장면은 수조 `1200 × 720`과 그 위의 외부 조명 영역을 합친 `1200 × 792`를 사용한다. 기본 진입은 이 장면 전체가 보이는 맞춤 보기이며, `100%`는 양옆 여백 없이 수조 폭이 화면을 채우는 cover 보기다.
- 수조 카메라 뷰포트는 창 전체를 채우고 상단·하단 HUD는 그 위에 뜬다. 화면비 차이는 종이색 레터박스가 아니라 수조 월드의 한 축을 조금 크롭해 처리한다.
- 카메라 컨테이너, 구조물, 충돌 도형, 생태 마스크, 포인터 변환에 같은 행렬을 사용한다.
- 어떤 각도로 회전해도 돌의 원본 종횡비와 면적이 시각적으로 변하지 않아야 한다.

수조의 시각적 스케일은 대략 `60 cm ≈ 1200 논리 픽셀`, 즉 `1 cm ≈ 20 픽셀`을 기준으로 삼는다. 이후 5 cm 물고기는 약 100픽셀, 2 cm 새우는 약 40픽셀 정도로 보이게 할 수 있다. 현미경적 조류 세포는 이 비율로 개체를 그리지 않고 군락 텍스처로 표현한다.

## 9. 광량 모델

### 9.1 하나의 광량장을 공유

성장 계산, 광량 탐침, 도감의 현재 위치 표시, 수조의 밝기 효과는 같은 캐시된 광량장을 사용한다. 화면만 밝고 실제 성장값은 어둡거나 그 반대인 상태를 허용하지 않는다.

### 9.2 상대 광량 계산

첫 두 미션에서는 실제 lux가 아니라 `0~100`의 상대적인 광합성 유효광량을 사용한다.

```text
국소 광량
= 광원 세기
 × 조사각 감쇠
 × 광원과의 거리 감쇠
 × 수심에 따른 물 감쇠
 × 구조물 가시성
+ 위쪽 개방도에 따른 산란광
+ 구조물 표면의 약한 1차 반사광
```

- 조명 바로 아래와 옆쪽이 확실히 다르게 느껴지는 광원 콘을 만든다.
- 물속 깊이에 따라 부드럽게 감쇠한다.
- 다른 돌이 광원과 표면 사이를 가리면 직사광이 줄어든다.
- 그림자도 완전한 0이 되지 않도록 약한 산란광 하한을 둔다.
- 자신의 앞면 광량을 잴 때 그 돌 자체를 가림 물체로 중복 판정하지 않는다.
- 천장 조명의 조사각은 수조 상단 좌우가 갑자기 0으로 끊기지 않을 만큼 넓게 잡되 중앙과 가장자리의 차이는 유지한다.
- 구조물의 위쪽 면은 받은 직접광 일부를 주변으로 한 번만 퍼뜨린다. 반사광은 재귀 계산하지 않으며 직접광보다 충분히 약하게 제한한다.
- 여러 광원이 생기면 기여도를 더한 뒤 0~100으로 정규화할 수 있다.

복잡한 파장별 흡수, 굴절, caustics와 체적 산란은 계산하지 않는다. 물 느낌을 위한 흔들리는 빛무늬는 시각 효과로 더할 수 있지만 생태 계산값을 뒤집어서는 안 된다.

### 9.3 계산 시점

- 구조물을 들고 이동 중: 저해상도 미리보기만 선택적으로 갱신
- 구조물이 낙하·회전 중: 정밀 광량 재계산 안 함
- 모든 영향 구조물이 안착: 정밀 광량장 한 번 계산
- 조명 설정 변경: 한 번 계산
- 생태 성장만 진행: 구조가 변하지 않는 한 기존 광량장 재사용

### 9.4 광량 탐침

탐침은 커서가 가리키는 실제 좌표를 측정하며, 가까운 임의 표면으로 조용히 달라붙지 않는다.

기본 표시:

```text
상대 광량 63 / 100
위치: 납작한 돌 앞면
```

종을 선택했을 때의 추가 표시:

```text
붓뚜껑말 예상 변화: ↑ 성장
현재 광량을 성장 곡선 위 점으로 표시
```

`부족/적정/과다`처럼 기준 종이 없는 전역 평가는 표시하지 않는다.

## 10. 조류 생태 모델

### 10.1 계산 단위

각 성장 가능 표면은 로컬 생태 셀의 집합을 가진다. 셀에는 다음 값이 들어간다.

- 종별 생체량 또는 점유 밀도 `0~1`
- 현재 상대 광량
- 표면 재질 적합도
- 이웃 셀과 접촉 구조물 정보
- 최근 순증감량

조류 개별 세포나 실 한 가닥을 물리 오브젝트로 만들지 않는다.

### 10.2 순성장

각 종의 생체량 변화는 같은 형태의 함수로 계산하되 계수와 광반응 곡선이 다르다.

```text
순변화
= 현재 생체량
 × (최대 성장률 × 종별 광 적합도 × 남은 수용력
    - 유지 비용
    - 저광량/강광 스트레스 사멸)
+ 이웃으로부터의 유입
- 이웃으로의 유출
```

중요한 규칙:

- 생체량이 수용력에 가까워지면 성장이 둔화한다.
- 불리한 광량에서는 실제 순변화가 음수가 되어 생체량이 감소할 수 있다.
- 사멸은 색이 옅어지고 면적이 줄어드는 모습으로 확인할 수 있어야 한다.
- 확산은 도착 셀의 서식 적합도가 양수일 때만 정착으로 이어진다.
- V1처럼 확산항이 항상 양수여서 부적합한 곳에서도 계속 커지는 현상을 허용하지 않는다.
- 두 종은 같은 셀의 수용력을 공유하므로 한 종의 증가가 다른 종이 쓸 공간을 줄인다.

### 10.3 확산

V2의 기본 확산은 다음 두 경로만 사용한다.

1. 같은 연속 표면의 8방향 인접 셀
2. 서로 실제로 맞닿거나 아주 가까운 돌·바닥재 표면 사이의 접촉 연결

수조 반대편으로 포자가 자동 순간이동하는 자유 수중 확산은 후속 기능으로 미룬다. 이 제한으로 접종 위치와 돌의 접촉 배치가 실제 선택이 된다.

### 10.4 종별 프로필

#### 붓뚜껑말속 — `Oedogonium sp.`

- 역할: 밝은 표면을 비교적 빠르게 점유하는 부착성 사상 녹조류
- 화면 표현: 가늘고 반투명한 녹색 털과 불규칙한 솜털형 군락
- 개별 실의 굵기는 수조 화면에서 과장하지 않고 1픽셀 안팎의 결로 표현
- 광반응: 중간~밝은 영역에서 빠르게 성장하고, 지나친 그늘에서는 유지 비용을 감당하지 못해 감소
- 도감 확대: 실제 사상 구조를 별도 그림으로 표시

#### `Nitzschia palea`

- 역할: 붓뚜껑말보다 그늘진 표면을 효율적으로 점유하는 부착성 규조류 계통
- 화면 표현: 황갈색·올리브색의 얇은 먼지막, 얼룩과 미세 점묘
- 개별 규조 세포는 수조 화면에 크게 그리지 않음
- 광반응: 낮은 광량에서도 양의 순성장이 가능하고, 밝은 곳에서는 즉시 죽지 않지만 경쟁상 이점과 순성장이 낮아질 수 있음
- 명칭: `어둠에서만 사는 조류`라고 설명하지 않고 `음영 적응형 부착성 규조류 계통`으로 표시

실제 종은 환경과 계통에 따라 반응 범위가 달라질 수 있으므로, 게임 데이터는 현실의 생태적 방향을 유지한 **수조 적응 계통의 단순화된 곡선**임을 도감에 밝힌다.

### 10.5 하나의 반응 곡선을 세 곳에서 사용

각 종의 광반응 데이터는 코드 한 곳에만 정의하고 다음 기능이 이를 읽는다.

- 실제 생체량 증가·감소
- 도감의 광반응 그래프
- 탐침의 `성장/유지/감소` 예측

UI용 별도 임계값은 만들지 않는다. `유지`는 현재 위치의 계산된 순변화 절댓값이 작은 구간으로 판정한다.

## 11. 조류 렌더링

### 11.1 수조 화면은 군락을 그린다

| 종 | 수조에서 보이는 것 | 수조에서 보이지 않는 것 |
|---|---|---|
| 붓뚜껑말속 | 얇은 녹색 솜털, 미세한 실 결, 군락 밀도 | 확대된 굵은 관 모양 실 한 가닥 |
| `N. palea` | 황갈색 먼지막, 점묘, 얇은 얼룩 | 쌀알만 한 개별 규조 세포 |

### 11.2 생태 셀과 화면 연결

- 각 생태 셀의 생체량을 마스크의 불투명도·색 농도·스탬프 밀도로 바꾼다.
- 셀 사각형이 보이지 않도록 이웃 값을 보간하고 가장자리를 불규칙하게 만든다.
- 붓뚜껑말의 미세한 흔들림은 절차적 변형으로 표현할 수 있지만 성장 위치를 바꾸지 않는다.
- 규조류는 표면 텍스처 위에 얇게 합성하며 큰 스프라이트를 반복하지 않는다.
- 같은 생체량이라도 완전히 똑같은 무늬가 반복되지 않도록 시드 기반 변형을 사용한다.

### 11.3 리깅의 위치

현재 체리새우는 몸통·꼬리·다리·더듬이 등 표시 부위를 나눈 절차형 2D 리그로 그린다. Worker가 보낸 위치·방향·행동을 보간하고, 렌더 프레임에서 부위별 작은 변형을 더한다. 이 애니메이션은 생태 좌표나 섭식 판정을 바꾸지 않는다.

추후 손그림 래스터 원화가 준비되면 같은 부위 계층과 변환점을 유지한 채 표시 레이어만 교체할 수 있다. 조류 군락은 개별 생물 리깅 대상이 아니며 생태 마스크 기반 절차 렌더링을 계속 사용한다.

## 12. 미션 설계

### 12.1 첫 번째 미션 — 빛을 찾는 붓뚜껑말

목표: 고정 조명 아래에서 납작한 돌과 붓뚜껑말 접종체를 배치해 제한시간 안에 지정된 돌 앞면의 목표 점유율을 만든다.

지급:

- 바닥재가 깔린 빈 수조
- 납작한 돌 1개
- 붓뚜껑말 접종체 1개
- 광량 탐침
- 고정 조명 1개

학습 순서:

1. 목록에서 돌을 꺼내 이동·회전한다.
2. 놓은 돌이 중력에 따라 안착하는 것을 본다.
3. 탐침으로 돌 앞면 여러 위치의 숫자를 비교한다.
4. 접종체를 적합한 위치에 놓는다.
5. 실행 후 군락이 퍼지고 부적합한 부분에서는 둔화·사멸하는 것을 본다.

판정 분모는 이 미션에서 지급한 **납작한 돌의 성장 가능 앞면**이다. 바닥재도 생태적으로는 성장 가능하지만 첫 미션 목표에는 포함하지 않아 학습 대상을 분명하게 한다.

성공 시 `Nitzschia palea`와 추가 구조물 두 종류를 해금한다.

### 12.2 두 번째 미션 — 밝은 곳과 그늘의 분업

목표: 붓뚜껑말과 `Nitzschia palea`를 함께 사용해 제한시간 안에 지급된 성장 가능 표면의 목표 비율을 점유한다.

지급:

- 납작한 돌, 둥근 돌, 세로로 긴 돌 각 1개
- 제한된 붓뚜껑말 접종체
- 제한된 `Nitzschia palea` 접종체
- 광량 탐침과 광량 오버레이
- 고정 조명 1개

플레이어는 구조물을 놓아 밝은 앞면, 부분 그늘과 어두운 바닥 영역을 만들고 두 종을 서로 다른 위치에 접종한다.

판정:

```text
점유율
= 두 종 중 하나가 점유 기준 밀도 이상인 셀의 유효 면적
 / 미션이 지정한 전체 성장 가능 면적
```

분모에는 세 돌의 앞면과 미션이 지정한 바닥재 영역을 포함한다. 지급된 돌은 모두 배치해야 시작할 수 있으므로 작은 돌 하나만 놓고 100%를 만드는 우회 해법을 막는다. 같은 셀에 두 종이 함께 있어도 면적을 두 번 세지 않는다.

정확한 접종체 수, 제한시간, 점유 기준 밀도와 목표 비율은 자동 시뮬레이션과 플레이 테스트 후 데이터 파일에서 조정한다.

### 12.3 공통 샌드박스 판정 원칙

- 미션은 구조물 종류·설치 개수·접종 위치 같은 특정 풀이를 정답으로 강제하지 않는다.
- 각 미션의 공개된 생태 목표만 만족하면 어떤 배치와 성장 경로를 사용해도 성공한다.
- 미량 생체량을 면적 목표로 세는 미션에서는 점유 기준 밀도를 적용한다.
- 실행 후 도전 모드의 배치는 잠긴다.
- 목표 상태가 짧게 스치기만 해도 성공하지 않도록 일정 시간 유지 조건을 둔다.
- 성공 또는 실패 뒤에도 시뮬레이션은 계속 진행한다.

### 12.4 네 번째 미션 — 첫 번째 소비자

네 번째 미션은 생산자 군락 위에 체리새우(`Neocaridina davidi`)를 추가해, 실제 먹이 섭취와 에너지 흐름만으로 성체 개체군을 유지하는 첫 소비자 과제다.

지급:

- 붓뚜껑말 접종체 4개
- `Nitzschia palea` 접종체 4개
- 체리새우 성체 4마리
- 세 종류 구조물 무제한
- 광량 탐침과 수온계

구조물, 조류와 새우는 허용 자원일 뿐 시작 필수 배치가 아니다. 아무것도 설치하지 않거나 일부만 설치한 상태에서도 시작할 수 있으며, 그 결과는 같은 섭식·에너지 규칙으로 자연스럽게 성공하거나 실패한다. 숨은 조류 비율, 구조물 수, 번식 횟수는 채점하지 않는다.

체리새우는 표면 셀의 실제 조류량을 탐색해 먹고, 먹은 만큼 붓뚜껑말과 규조류 생체량을 줄이며 에너지를 얻는다. 에너지는 활동과 시간 경과로 감소한다. 어린 개체는 먹이가 있을 때 성장하고, 성체 암수는 에너지·먹이·근접 조건을 만족하면 포란과 출산을 거쳐 번식한다. 에너지가 고갈되거나 개체별 압축 수명에 도달하면 사망하므로 먹이가 충분한 장기 수조에서도 세대가 교체된다.

성공 판정:

```text
체리새우 성체 수 >= 4
상태를 연속 120초 유지
제한시간 300초
```

어린 개체는 성체가 된 뒤부터 목표 수에 포함하고, 유지 중 성체 수가 4마리 아래로 내려가면 연속 유지 시간을 다시 센다. 성공·실패 뒤에도 새우와 조류의 변화는 계속된다.

산소 생산·소비, 배설과 사망 질량은 공간별 ledger에 기록하지만 현재는 새우 생존·번식·미션 판정에 영향을 주지 않으며 UI에도 표시하지 않는다.

## 13. 실험실

실험실은 부가 장식 모드가 아니라 시뮬레이션 규칙을 이해하는 핵심 기능이다. 현재 네 미션 범위에서 다음을 조절할 수 있다.

- 해금된 세 종류 구조물의 수와 위치·회전
- 붓뚜껑말과 `N. palea`의 접종 수와 위치
- 체리새우의 투입 수와 위치
- 고정 조명의 위치, 방향, 세기
- 시간 배속
- 광량 오버레이
- 광량 탐침과 종별 성장 예측
- 시뮬레이션 일시정지, 재개와 초기화

구조물, 두 조류와 체리새우는 무제한으로 꺼낼 수 있다. 도전 미션과 실험실은 같은 물리·광량·성장·섭식·에너지·생애 주기 코어와 같은 종 데이터를 사용하며, 실험실만을 위한 더 쉬운 규칙은 만들지 않는다. 실험실에는 성공·실패 목표가 없고, 실행 중 편집은 일시정지 상태에서만 허용한다. 관찰 전용 생지화학 ledger도 같은 방식으로 기록하지만 현재 일반 UI에는 표시하지 않는다.

## 15. 성능 구조

### 15.1 작업 빈도 분리

| 작업 | 권장 실행 시점 |
|---|---|
| 포인터에 든 물체 표시 | 렌더 프레임마다, 변환만 변경 |
| 동적 돌 물리 | 고정 시간 간격, 최대 서브스텝 제한 |
| 정밀 광량 레이 계산 | 구조물 안착 또는 조명 변경 후 한 번 |
| 조류 성장 | 물리보다 낮은 고정 주기 |
| 체리새우 이동·조향 | 30Hz 고정 시뮬레이션 스텝 |
| 체리새우 섭식·에너지·생애 주기 | 조류 성장과 같은 0.25초 생태 틱 |
| 체리새우 리그 애니메이션 | 렌더 프레임마다 상태 보간과 부위 변형 |
| Worker 상태 전달 | 전체 복사가 아닌 변경량 중심 |
| 광량 화면 효과 | 캐시된 텍스처 재사용 |

### 15.2 반드시 제거할 V1 패턴

- 매 상태 갱신마다 모든 구조물 표시 객체를 파괴하고 재생성
- 돌이 움직이는 매 프레임마다 전체 광량 레이캐스팅
- 광량 셀마다 별도 사각 그래픽 객체 생성
- 화면 프레임률에 따라 물리 결과가 달라지는 가변 시간 적분
- 변하지 않은 대형 상태 배열을 매번 Worker와 복사

### 15.3 V2 방식

- 구조물과 군락 표시 객체를 유지하고 위치·각도·텍스처만 갱신한다.
- 체리새우 표시 객체도 개체 ID별로 재사용하고 Worker 스냅샷 사이의 위치를 보간한다.
- 체리새우마다 Matter.js 강체를 만들지 않고 연속 좌표·속도와 조향 규칙으로 이동시킨다.
- 광량장은 하나의 저해상도 버퍼/렌더 텍스처로 합성한다.
- 시뮬레이션 Worker는 더티 영역이나 변경된 개체만 전달한다.
- 물리는 고정 스텝과 누적 시간을 사용하며 한 프레임의 최대 보정 횟수를 제한한다.
- 잠든 돌은 물리와 광량 계산에서 제외한다.
- 개발 빌드에 프레임 시간, 물리 시간, 광량 계산 시간과 생태 계산 시간을 표시하는 진단 패널을 둔다.

성능의 1차 수용 기준은 개발 중인 낮은 사양 Mac에서 **돌 하나가 낙하할 때 눈에 띄는 멈춤이 없는 것**이다. 이후 10개 구조물과 두 종의 전체 표면 군락을 놓은 표준 장면의 프레임 시간과 계산 시간을 기록해 회귀 기준으로 사용한다.

## 16. 데이터 분리

다음 값은 UI나 시뮬레이션 코드에 흩어 넣지 않고 데이터 파일로 관리한다.

## 17. 구현 순서

### 1단계 — 좌표와 구조물 기하

- 비등방 스케일 제거와 레터박스 적용

## 18. 구현 수용 기준

### 구조물과 입력

- [ ] 돌을 0도, 45도, 90도로 회전해도 그림 비율이 변하지 않는다.
- [ ] 디버그 충돌선이 돌 그림의 외곽과 육안상 일치한다.
- [ ] 생태 마스크가 돌 앞면을 덮고 투명한 바깥으로 나가지 않는다.
- [ ] 한 번 클릭해 집고 마우스를 놓은 상태로 이동한 뒤 다시 클릭해 놓을 수 있다.
- [ ] 회전 버튼을 연타하지 않아도 원하는 각도를 빠르게 만들 수 있다.
- [ ] 접종체 수량이 0일 때 기존 접종체가 자동 이동하지 않는다.

### 물리와 광량

- [ ] 기울어진 돌은 놓은 뒤 토크를 받아 자연스럽게 회전한다.
- [ ] 안정된 돌은 잠들고, 지지대가 사라지면 다시 떨어진다.
- [ ] 같은 깊이라도 조명 바로 아래와 옆쪽의 측정값이 다르다.
- [ ] 다른 돌로 가리면 뒤쪽 표면의 광량이 줄어든다.
- [ ] 물리 낙하 중 정밀 광량 계산 때문에 화면이 멈추지 않는다.

### 생태와 표현

- [ ] 붓뚜껑말은 적합한 빛에서 증가하고 심한 그늘에서 눈에 띄게 감소한다.
- [ ] `N. palea`는 그늘에서 붓뚜껑말보다 유리하지만 밝은 곳에 놓자마자 죽지 않는다.
- [ ] 탐침의 예상 증감 방향과 다음 시뮬레이션 구간의 실제 증감 방향이 일치한다.
- [ ] 부적합한 셀에서는 이웃 확산만으로 군락이 무한히 증가하지 않는다.
- [ ] 조류가 돌의 외곽선 한 줄이 아니라 앞면 내부에 군락으로 보인다.
- [ ] 규조류가 물고기 크기의 개별 타원으로 보이지 않는다.

### 미션과 실험실

- [ ] 첫 미션은 돌 위치와 접종 위치에 따라 성공·실패가 달라진다.
- [ ] 두 번째 미션은 규조류만 사용할 수 있으며, 구조물로 적정 그늘을 만들지 않으면 목표 점유가 오르지 않는다.
- [ ] 네 번째 미션은 구조물·조류·새우를 의무 설치하지 않아도 시작할 수 있다.
- [ ] 체리새우가 먹은 셀의 실제 조류 생체량이 섭취량만큼 감소하고, 섭취량에 따라 에너지가 증가한다.
- [ ] 먹이가 없는 조건에서는 초기 성체 4마리를 연속 120초 유지하지 못하고 자연스럽게 실패한다.
- [ ] 충분한 먹이가 있는 조건에서는 성체 4마리를 연속 120초 유지해 제한시간 300초 안에 성공할 수 있다.
- [ ] 태어난 어린 개체는 성장해 성체가 된 뒤에만 목표 성체 수에 포함된다.
- [ ] 사망은 해당 위치의 고형 유기물 ledger에 한 번만 기록되고, 같은 위치에 보이는 사체를 만들지만 아직 수질 피해는 만들지 않는다.
- [ ] 관찰 전용 ledger를 켜지 않은 현재 설정에서 산소·배설·사망 질량이 생존·성장·번식·미션 결과를 바꾸지 않는다.
- [ ] 도전 모드 실행 후 배치는 잠긴다.
- [ ] 실험실은 체리새우와 두 조류를 무제한으로 제공하고, 일시정지 중 같은 시스템 범위의 편집을 허용한다.
- [ ] 종 카드에서 특성, 크기, 군락 형태와 광반응을 확인할 수 있다.

## 20. 참고 자료와 표현 원칙

- `Nitzschia palea`는 다양한 광환경에서 광순응을 보일 수 있으므로, 게임에서는 실제 종을 `어둠에서만 사는 종`으로 단정하지 않는다. [광순응 연구](https://www.sciencedirect.com/science/article/abs/pii/S0304377007001726)
- `Nitzschia`류가 부착 생물막에서 관찰되는 점은 황갈색 표면막 표현의 근거로 삼되, 현실의 모든 계통을 같은 성질로 일반화하지 않는다. [부착 생물막 연구](https://www.sciencedirect.com/science/article/pii/S092585742200204X)
- 붓뚜껑말속도 계통과 환경에 따라 광반응이 다를 수 있으므로, 도감에는 게임용 계통의 곡선임을 밝힌다. [Oedogonium 광반응 연구](https://li01.tci-thaijo.org/index.php/JFE/article/view/228524)

현실 자료는 방향과 설명을 정하는 근거이며, 미션 정답을 만들기 위해 실제 종을 거짓으로 설명하지 않는다. 게임에 필요한 보정은 `수조 적응 계통`, `상대 광량`, `압축된 게임 시간`처럼 사용자에게 이해 가능한 형태로 공개한다.

## 21. 2차 사용성·생태 모델 수정안

이 절은 V2 실행본 플레이 피드백을 반영한 우선 수정 기준이며, 앞 절과 충돌하면 이 절을 따른다.

### 21.1 가독성과 수조 비율

- 기능성 본문은 15px, 보조 설명은 13px, 버튼과 탭은 14px 이상을 기본으로 한다.
- 12px 미만의 정보 글씨는 사용하지 않는다.
- 오뮤 다예쁨체는 제목과 짧은 강조에 사용하고, 긴 설명·계측값·그래프는 시스템 고딕을 사용한다.
- 수조의 논리·표시 비율은 항상 `1200:720(5:3)`으로 고정한다.
- 창 크기가 달라져도 수조를 비등방으로 늘리지 않으며, 부족한 패널 공간은 스크롤 또는 접기 방식으로 처리한다.

### 21.2 입력 상태

- `선택/관찰`이 항상 기본 상태다. 이 상태에서 클릭은 정보를 선택할 뿐 구조물을 움직이지 않는다.
- `이동/배치`를 명시적으로 선택했을 때만 기존 구조물을 집는다.
- 광량 측정은 한 지점을 클릭해 값을 고정하는 일회성 도구로 동작하고 곧바로 선택 상태로 돌아간다.
- 같은 도구 버튼, `Esc`, 우클릭으로 모든 임시 도구를 해제할 수 있다.
- 실행 중 특정 도구를 자동 활성화하지 않는다.
- 인벤토리 항목은 DOM 커서 미리보기로 목록에서 꺼낸 뒤, 수조에 들어왔을 때만 월드 배치 미리보기를 만들고 클릭 시 실제 오브젝트로 확정한다.
- 회전은 휠과 `Q/E`를 주 조작으로 하며, 접근성용 버튼은 수조 아래의 간결한 배치 막대에 둔다.

### 21.3 상황별 정보

- 아무 대상도 선택하지 않았으면 우측에 종 반응 곡선을 임의로 표시하지 않는다.
- 구조물 선택 시 구조물 정보와 해당 위치 환경을, 군락 선택 시 우점종·혼합 비율·광량·수온·성장 예상을 표시한다.
- 수조에서 군락을 선택하면 그 종의 반응 곡선에 현재 환경 마커를 표시한다.
- 인벤토리의 정보 열람과 수조에 실제 존재하는 대상 선택을 구분한다.

### 21.4 접종·확산·경쟁

- 접종 표시는 배치 단계에서만 편집 표식으로 사용하고 실행과 동시에 제거한다.
- 생태 셀은 계산 단위일 뿐 렌더링 격자로 노출하지 않는다. 군락은 보간된 연속 밀도장과 저주파 변이로 그린다.
- 낮은 생물량도 희미한 점·막·솜털로 인식할 수 있도록 비선형 시각 보정을 사용한다.
- 확산은 군락이 주변에 번식체 압력을 만들되, 실제 정착량을 출발 군락에서 차감해 같은 양을 도착 셀에 더하는 질량 이동으로 계산한다.
- 여러 군락이 같은 빈 공간을 노리거나 한 군락이 여러 이웃으로 퍼질 때는 도착 공간과 출발 생체량을 기준으로 이동량을 비례 제한한다.
- 가까운 표면과 접촉한 구조물로 먼저 퍼지며, 광량·수온·기질 적합도가 높은 셀의 정착 확률이 높다.
- 포화된 셀에도 자연 탈락과 재정착을 적용해 환경에 더 적합한 종이 점유 비율을 서서히 바꿀 수 있게 한다.
- 붓뚜껑말이 발달하면 아래의 낮은 규조류 막을 일부 차광하는 단순화된 상층 효과를 적용한다.
- 재접종은 즉시 상대 종을 삭제하지 않고 해당 종의 국소 번식 압력을 높인다.

### 21.5 광량과 수온

- 불투명 구조물에 막힌 직접광의 고정 최소 투과율을 제거한다.
- 광원 면적 샘플로 반그림자를 만들고, 산란광은 고정 바닥값이 아니라 위쪽으로 열린 정도에 따라 계산한다.
- 구조물을 겹칠수록 차광이 누적되며, 폐쇄된 틈은 조류가 사멸할 정도로 어두워질 수 있어야 한다.
- 첫 범위의 수온은 잘 섞인 수조의 전역 상태로 계산한다. 실내 온도와 조명 발열을 따라 천천히 변한다.
- 온도계는 실제 °C를 표시하고, 종별 순성장률은 광량 반응과 넓은 수온 적합도 곡선을 함께 사용한다.
- 초기 미션의 수온은 관찰 가능한 변수이되 사용자가 알 수 없는 실패 함정이 되지 않게 조정한다.

### 21.6 세 번째 미션 — 닿지 않는 빛

- 세 번째 미션 자체에는 새우를 투입하지 않고 제한된 빛 속에서 붓뚜껑말 증식을 실험한다.
- 빈 수조와 제한된 접종 횟수, 선택 가능한 구조물들을 지급한다.
- 성공은 수조 전체에서 자란 붓뚜껑말의 총량만으로 판정한다.
- 위치·높이·구조물 종류·광량·접촉 연결 여부는 성공 조건에 직접 포함하지 않는다.
- 구조물 배치와 직사광·산란광·반사광은 붓뚜껑말의 성장 환경을 바꾸는 원인일 뿐, 특정 풀이를 강제하는 점수 규칙이 아니다.

### 21.7 결과 이후의 계속 관찰

- 미션 판정(`진행 중/성공/실패`)과 시뮬레이션 재생 상태(`재생/일시정지`)를 분리한다.
- 성공 또는 제한시간 실패 뒤에도 물리·성장·사멸·경쟁 계산을 계속한다.
- 결과는 화면을 막는 팝업 대신 상단 배지와 비차단 알림으로 표시한다.
- 성공과 실패는 판정 당시 기록으로 고정한다. 이후 군락 상태는 별도의 `현재 조건 충족 여부`로 표시한다.
- 성공 여부와 관계없이 네 미션은 처음부터 모두 선택할 수 있다. 도전 모드의 배치 잠금은 각 미션 내부 규칙으로만 유지한다.
- 결과 이후에도 배속, 일시정지, 다시 도전을 사용할 수 있다.

### 21.8 추가 수용 기준

- [ ] 1280×720과 더 큰 창에서 기능성 텍스트가 12px 미만으로 내려가지 않는다.
- [ ] 창 크기를 바꿔도 수조의 5:3 비율과 내부 오브젝트 비율이 유지된다.
- [ ] 기본 선택 상태에서 클릭만으로 구조물이 움직이지 않는다.
- [ ] 광량 측정 한 번 뒤 선택 상태로 돌아가며 `Esc`로 측정을 취소할 수 있다.
- [ ] 인벤토리 항목이 수조 중앙에서 먼저 생성되지 않고 포인터에서 꺼내지는 것으로 보인다.
- [ ] 시뮬레이션 시작 뒤 접종 핀이 사라진다.
- [ ] 포화된 규조류 군락도 붓뚜껑말에 유리한 조건과 재접종이 주어지면 점유 비율이 서서히 역전된다.
- [ ] 희박한 조류가 보이면서도 셀 중심의 바둑판무늬는 드러나지 않는다.
- [ ] 돌 하나 뒤보다 두 개가 겹친 뒤의 광량이 낮고, 폐쇄된 틈은 붓뚜껑말 유지 광량보다 낮아진다.
- [ ] 성공·실패 판정 뒤에도 생태계가 계속 변화하고 일시정지·배속 조절이 가능하다.
- [ ] 세 번째 미션은 붓뚜껑말이 어디에서 자랐는지와 무관하게 수조 전체 총량이 같으면 같은 진행도로 판정한다.

### 21.9 메인 화면 진행 기록

- 네 미션은 처음부터 모두 열려 있으며, 성공한 미션 ID는 표시용 로컬 완료 기록으로만 저장한다.
- 완료한 미션 카드 우하단에는 두들 스타일의 `✓ 완료` 도장을 표시한다.
- 이전 버전의 순차 해금 기록은 완료 도장 마이그레이션에만 사용하고 이후 접근 제한에는 사용하지 않는다.
- 메인 우상단에는 텍스트가 함께 있는 `설정` 버튼을 둔다.
- 설정창에서는 완료 수와 미션별 완료 여부를 확인할 수 있다.
- `미션 완료 기록 초기화`는 확인 단계를 거친 뒤 완료 도장만 지운다. 초기화 뒤에도 네 미션과 실험실은 모두 선택할 수 있다.
- 초기화는 실험실 접근이나 다른 앱 설정에는 영향을 주지 않는다.

## 22. 3차 미션·분석 UI 수정안

이 절은 실제 플레이 피드백을 반영한 최신 구현 기준이다. 특히 21.2의 일회성 측정 방식과 21.6의 세 번째 미션 설계를 대체한다.

### 22.1 미션 안내와 진행 표시

- 도전 과제에 들어가면 시뮬레이션 화면보다 먼저 셀랩식 실험 의뢰서를 표시한다.
- 의뢰서에는 실험 질문, 지급 자원, 정량 목표와 성공 판정만 적는다.
- 구조물을 어디에 어떻게 놓을지, 어떤 수치를 찾아야 할지 같은 공략 순서와 정답 힌트는 제공하지 않는다.
- 닫은 뒤에도 우측 `미션 설명` 버튼으로 같은 내용을 다시 열 수 있다.
- 상단에는 남은 시간 대신 `실행 시간 / 제한 시간`을 표시한다.
- 미션 진행도는 퍼센트만이 아니라 실제 판정 단위도 함께 표시한다. 면적 목표는 `%`, 세 번째 미션은 `현재 붓뚜껑말 총량 / 목표 총량`을 사용한다.
- 네 번째 미션은 `현재 성체 수 / 목표 성체 수`와 `연속 유지 시간 / 120초`를 함께 표시한다.
- 세 번째 미션에는 별도의 위치별 점수 칸이나 목표 위치 표식을 두지 않는다.
- 내부 계산 격자는 생태 시뮬레이션에만 사용하고, 사용자가 특정 위치를 정답으로 오해할 수 있는 채점 표시는 노출하지 않는다.

### 22.2 두 번째 미션 — 규조류를 위한 그늘

- 접종 가능한 생물은 음영 적응형 규조류 `Nitzschia palea` 하나뿐이다.
- 구조물 지급량은 넓적한 사암 3개, 둥근 강돌 4개, 세로 판석 3개로 늘린다.
- 시작에 필요한 고정 구조물 종류나 개수는 없다. 다만 목표가 집계될 구조물 앞면은 하나 이상 마련하고 그 앞면에 규조류를 접종해야 한다.
- 사용자에게는 이를 `안정된 규조류 면적`이라고 부른다. 내부 성공 점수에는 구조물 앞면 중 광량 `12~58`, 규조류 생물량 `0.045` 이상인 셀만 포함한다.
- 위 조건을 만족하는 셀이 전체 목표 구조물 표면의 `18%` 이상인 상태를 4초 유지하면 성공한다.
- 바닥재와 너무 밝은 표면의 규조류는 살아 있을 수 있으나 미션 진척도에는 포함하지 않는다.

### 22.3 세 번째 미션 — 닿지 않는 빛

- 바닥의 고정 구조물과 고정 접종 군락을 제거한다. 빈 수조에서 시작한다.
- 넓적한 사암·둥근 강돌·세로 판석은 모두 무제한으로 제공하며, 특정 종류나 개수를 의무로 설치하지 않는다. 목표 환경을 만드는 데 필요한 구조물만 골라 쓸 수 있다.
- 성공 판정에는 수조의 모든 표면과 바닥재에서 자란 붓뚜껑말 생물량을 전부 합산한다.
- 합산한 붓뚜껑말 총량이 `145` 이상인 상태를 5초 유지하면 성공한다.
- 성장 위치·높이·해당 위치의 광량·구조물 종류·연결 관계·직사광 여부는 채점 조건에 넣지 않는다.
- 사용자는 구조물을 쌓거나, 반사광과 산란광을 이용하거나, 다른 배치를 고안하는 등 원하는 방식으로 성장 환경을 만들 수 있다.
- 기본 광원 출력은 `60`으로 낮춘다. 이는 미션의 환경 제약이며 특정 배치를 요구하는 숨은 성공 조건이 아니다.

### 22.4 광량 탐침과 수온계

- 광량 탐침 또는 수온계를 고르면 포인터 위치의 광량·수온·표면 이름을 실시간으로 미리 표시한다.
- 수조를 클릭하면 그 위치에 측정점이 설치되고 도구는 기본 선택 상태로 돌아간다.
- 측정점은 여러 개 설치할 수 있으며 시뮬레이션 중 값이 계속 갱신된다.
- 측정 탭에서 설치 목록을 비교하고, 수조의 측정점을 클릭하면 해당 측정점 정보만 우측에 표시한다.
- 측정점은 정보 패널에서 개별 회수할 수 있다.

### 22.5 탭별 선택 문맥

- 구조물 탭의 선택은 구조물만, 생물 탭의 선택은 군락만, 측정 탭의 선택은 설치된 측정점만 감지한다.
- 생물 탭에서 한 지점을 클릭하면 그 셀에 실제로 존재하는 종만 정보 탭으로 표시한다.
- 생물 탭에서 드래그하면 사각 영역 안의 군락을 합산하며, 그 영역에 존재하는 종만 전환 탭으로 표시한다.
- 체리새우를 한 마리 클릭하면 해당 개체의 생활 단계·성별·행동·에너지·번식 상태를 표시한다.
- 생물 탭에서 드래그한 영역 안에 체리새우가 있으면 군락 정보와 별도로 선택된 새우의 성체·어린 개체 수와 평균 에너지를 집계한다.
- 아무것도 선택하지 않았을 때는 현재 탭에서 무엇을 선택할 수 있는지 안내하며 임의의 생물 정보나 성장 곡선을 표시하지 않는다.

### 22.6 배치와 렌더 보정

- 구조물을 들고 있을 때 회전 버튼은 화면 아래 막대가 아니라 구조물 좌우에 원형 화살표로 표시한다.
- 한 번 누를 때 10도 회전하며 길게 눌러 연속 회전할 수 있다. 휠과 Q/E 조작도 유지한다.
- 바닥 조류는 바닥재 윗선 뒤에서 자라도록 마지막 전경 레이어에 바닥 경계선을 다시 그려 겹침을 정리한다.
- 미션 성공·실패 뒤에도 시뮬레이션은 계속 진행하며 상단 기록과 현재 조건 상태를 분리해 보여준다.

### 22.7 추가 수용 기준

- [x] 미션 진입 시 설명창이 열리고 실행 중에도 다시 볼 수 있다.
- [x] 상단에서 실행 시간과 목표 진행률을 동시에 확인할 수 있다.
- [x] 두 번째 미션에서 붓뚜껑말은 접종할 수 없고 규조류만 목록에 보인다.
- [x] 세 번째 미션은 빈 수조에서 시작하며 상부 직접 접종으로 자동 성공 테스트를 통과한다.
- [x] 탐침과 수온계가 포인터를 따라 미리보기 되고 클릭 위치에 여러 측정점을 설치한다.
- [x] 점 선택과 영역 선택이 현재 탭 문맥에 따라 다른 대상만 반환한다.
- [x] 한 종만 있는 위치를 선택하면 존재하지 않는 종 탭이 나타나지 않는다.

## 23. 수조 중심 HUD·카메라·광원 보정

이 절은 패널을 항상 노출하던 이전 화면 구성과 탭별 선택 문맥을 대체한다.

### 23.1 수조 중심 화면

- 기본 화면에서는 수조가 사용 가능한 중앙 영역 대부분을 차지한다.
- 메뉴, 보유 목록, 퀘스트와 관찰 기록은 모두 기본 닫힘이며 서로 독립적으로 열고 닫는다.
- 패널은 창 레이아웃을 밀어내지 않는 종이·포스트잇 오버레이로 표시한다. 사용자가 연 패널만 수조를 일부 덮는다.
- 좌측에는 메뉴와 보유 목록, 우측에는 퀘스트와 관찰 기록을 쌓아 같은 쪽 패널끼리 완전히 겹치지 않게 한다.
- 미션 설명창은 미션 진입 시 유지하고, 실행 중에는 메뉴 또는 퀘스트 포스트잇에서 다시 연다.
- 미션 목록 복귀는 햄버거 메뉴 안에만 숨기지 않고 상단 좌측에 항상 보이는 버튼으로 둔다.
- 인벤토리에서 항목을 꺼내면 패널을 닫고 항목이 커서에 붙는다. 그 다음 수조의 유효 위치를 한 번 클릭하면 바로 놓여야 한다.

### 23.2 카메라 조작

- 수조 논리 좌표 `1200 × 720`은 그대로 유지한다. 카메라는 같은 폭의 상단 외부 조명 영역을 합친 `1200 × 792` 장면 경계를 사용한다. 맞춤 보기에서는 전체 장면을 보이고, `100%`에서는 수조가 창의 양옆을 채우며 한 축만 자연스럽게 크롭한다.
- 상단·하단 HUD 뒤까지 의도된 장면 배경이 이어지며 확대·이동 중 종이색 레이아웃 여백이나 물색 외부 공간이 노출되어서는 안 된다.
- 수조는 `0…1200 × 0…720`, 고정 광원은 음수 Y의 별도 형제 레이어에 둔다. 둘은 같은 카메라 아래에서 함께 확대·이동하지만 광원은 유리·물 마스크 안에 포함되지 않는다.
- 보이는 광원의 위치·폭은 광량 계산이 읽는 공용 광원 설정을 사용한다. 화면용으로 따로 맞춘 광원이 실제 조사 위치와 어긋나서는 안 된다.
- 수조 내용은 둥근 유리 안쪽 마스크로 자르고, DOM 배경은 투명하게 둔다. 확대·이동 뒤에도 고정된 청록색 직사각형이나 이중 테두리가 남지 않아야 한다.
- 카메라 버튼과 수치 패널만 화면 HUD로 고정하고, 어항의 일부인 장식은 화면에 고정하지 않는다.
- `100%`는 수조 폭이 창을 채우는 장면 cover 기준이다. 축소 하한은 창 비율에 따라 `contain / cover`로 계산하며, 이 맞춤 배율에서는 조명 전체와 수조 프레임 네 변이 모두 보인다.
- 확대 범위는 동적 전체 보기 배율부터 `400%`까지이며 휠 확대는 포인터 아래의 월드 위치를 고정한다. 전체 보기 바깥은 상단·측면 모두 동일한 저채도 회녹색 외부 공간으로 처리한다.
- 맞춤 보기에서는 카메라 조작을 수조 왼쪽 외부 여백에 세로로 두고, 확대 보기에서는 우하단에 둔다. 확대·축소 버튼은 cover 기준인 `100%`를 건너뛰지 않는다.
- 이동 모드에서는 수조를 드래그해 카메라를 옮긴다. 구조물을 들고 있을 때는 이동 모드를 잠그고 휠을 구조물 회전에 사용한다.
- 배치, 점 선택, 영역 선택과 측정은 모두 동일한 카메라 역변환을 거쳐 월드 좌표로 전달한다.
- 수조의 상단 공기층이나 월드 바깥을 클릭해도 가장 가까운 가장자리로 강제 접종·측정되지 않는다.
- 클릭과 영역 드래그의 구분은 월드 거리가 아니라 화면 픽셀 거리로 판정해 배율에 따라 조작감이 달라지지 않게 한다.
- 다시 도전하거나 다른 미션에 들어가면 카메라는 조명과 수조를 모두 포함하는 동적 맞춤 보기로 초기화한다.
- 배치 중 상태·취소 조작은 바닥과 구조물을 덮지 않도록 오브젝트를 둘 수 없는 상단 공기층 쪽에 표시한다.

### 23.3 패널이 닫힌 상태의 선택

- 보유 목록 탭은 수조 선택 필터로 쓰지 않는다. 패널이 닫힌 뒤에도 최근 탭 때문에 엉뚱한 대상이 선택되어서는 안 된다.
- 기본 선택은 클릭 위치의 측정점, 군락, 구조물 순으로 실제 존재하는 가장 적절한 대상을 고른다.
- 선택 결과는 관찰 기록 아이콘의 상태로 알리고, 사용자가 관찰 기록을 열었을 때 해당 대상 정보만 표시한다.

### 23.4 광원 보정 범위

- 천장 광원의 반각은 수조 폭 대부분에 직접광이 닿도록 넓히되 가장자리는 중앙보다 어둡게 유지한다.
- 산란광은 고정 최소값이 아니라 수면 쪽으로 열린 정도와 수심을 사용한다. 따라서 개방된 가장자리는 0으로 끊기지 않고 폐쇄된 틈은 충분히 어두워질 수 있다.
- 직접광을 받은 구조물 위쪽 표면에서 약한 1차 반사광을 만든다. 이 반사광도 다른 구조물에 가려지며 재귀 반사는 계산하지 않는다.
- 빈 수조의 좌우 상단과 그 아래 지점이 모두 0보다 크고, 구조물 뒤 핵심 그림자는 낮은 광량을 유지하는 자동 테스트를 둔다.

## 24. 첫 소비자 구현 기준

이 절은 네 번째 미션과 체리새우가 추가된 현재 구현의 최신 기준이다. 앞 절의 생산자 전용 설명과 충돌하면 이 절을 따른다.

### 24.1 미션 4 구성과 판정

- 이름: `첫 번째 소비자`
- 제한시간: 300초
- 목표: 체리새우 성체 4마리를 연속 120초 유지
- 지급: 체리새우 성체 4마리, 붓뚜껑말·규조류 접종체 각 4개, 세 종류 구조물 무제한, 광량 탐침과 수온계
- 시작 조건: 필수 구조물·필수 접종·필수 새우 배치 없음
- 채점 조건: 성체 수와 연속 유지 시간만 사용하며 조류량, 구조물 수와 번식 횟수는 직접 채점하지 않음
- 성체 수가 4마리 아래로 내려가면 유지 시간을 0부터 다시 계산
- 어린 개체는 성체로 성장한 뒤부터 목표 수에 포함
- 성공 또는 제한시간 실패가 기록된 뒤에도 시뮬레이션은 계속 진행

네 미션과 실험실은 처음부터 모두 접근할 수 있다. 완료 기록은 카드의 표시와 설정의 초기화 대상일 뿐, 다음 미션 접근을 잠그지 않는다.

### 24.2 체리새우 이동 AI

체리새우는 Matter.js 강체가 아니라 위치·속도·방향을 가진 운동학적 개체다. 이동은 기본 30Hz 고정 스텝으로 계산하고 렌더러가 Worker 스냅샷 사이를 보간한다. 장기 관찰용 32·64배속에서는 계산량을 제한하기 위해 0.1초 조향 스텝으로 묶되 화면 리그 보간은 계속 렌더 프레임마다 수행한다. 고배속의 조향과 생태 틱은 각 단계의 예정 시각 순서대로 교차 실행해, 이동 전 위치에서 미래의 섭식을 한꺼번에 처리하지 않는다.

행동 상태는 `들고 있음`, `탐색`, `먹이로 이동`, `섭식`, `휴식`, `굶주림`으로 나뉜다. 평상시에는 현재 위치 반경 `64` 안에서, 에너지가 `0.35` 이하인 약한 개체만 반경 `520` 안에서 먹이 목표를 찾는다. 접종점은 별도 목표로 취급하지 않으며, 성장·확산한 셀도 같은 규칙으로 후보가 된다. 후보는 다음 값을 함께 고려해 결정적으로 고른다.

- 셀에 실제로 남은 섭취 가능 조류량
- 현재 위치에서 셀까지의 거리
- 같은 먹이를 향하는 개체의 혼잡도
- 이전 목표 유지와 개체별 시드 노이즈

에너지가 `0.48` 이하가 되면 먹이 탐색을 시작하고, 섭식으로 `0.70`까지 회복할 때까지 먹이 탐색 상태를 유지한다. 한 번의 섭식 세션은 3~5초 또는 누적 섭취 생체량 `1.4` 중 먼저 도달한 때 끝난다. 충분히 먹고 세션이 끝난 개체는 별도의 군락 기억 없이 기존 행동 타이머만 사용해 2.5~4초 동안 먹이 탐색을 중단하고 주변으로 이동한 뒤 다시 탐색한다. 굶주림 단계의 개체에는 이 강제 이탈을 적용하지 않는다.

가까운 개체끼리는 약한 분리 조향을 적용하고 수조 경계를 벗어나지 않게 위치를 제한한다. 탐색 목적지는 먹이가 과밀한 셀만 반복 선택하지 않도록 분산하며, 현재 AI는 자연스러운 탐색과 군집 겹침 완화를 위한 단순화된 조향이다. 돌 충돌을 푸는 경로 탐색·유체 흐름·다리별 접촉 물리는 구현하지 않는다.

### 24.3 실제 섭식과 에너지

규조류와 붓뚜껑말의 실제 섭취 가능 생체량을 먹이 탐색과 섭식 비율에 반영한다. 두 종을 합산한 먹이 가치가 `0.04` 이상인 셀만 먹을 수 있다. 여러 새우가 같은 셀을 동시에 먹으면 요청량을 합산한 뒤 셀에 실제로 남은 양을 한도로 각 개체의 섭취량을 비례 배분한다. 기본 섭식률은 시뮬레이션초당 `0.375`이며, 셀의 먹이 밀도에 따라 실제 요청량이 조절된다.

섭취가 확정되면 해당 셀의 두 조류 생체량을 실제 섭취량만큼 줄이고, 새우 에너지는 `실제 섭취량 × 0.08`만큼 증가한다. 개체별 누적 섭식량과 수조 전체 누적 섭식량도 같은 실제 감소량으로 기록한다. 에너지는 성체·어린 개체의 기초 대사와 이동 상태에 따른 활동 비용으로 계속 감소하며, 0이 되면 개체가 사망한다. 따라서 조류가 없거나 접근 가능한 먹이가 고갈된 수조는 별도의 실패 스크립트 없이 같은 규칙으로 무너진다.

### 24.4 성장과 번식

- 초기 지급 성체는 암수가 번갈아 생성된다.
- 어린 개체는 에너지 `0.44` 이상이고 최근 12초 안에 먹은 상태에서만 성장 진행도가 쌓인다.
- 나이와 먹이를 확보한 성장 진행도가 모두 180초 조건을 충족하면 성체가 된다.
- 암컷은 에너지 `0.46`, 수컷은 `0.45` 이상이어야 하며, 양쪽 모두 최근 8초 안에 실제로 섭식해야 교미 후보가 된다.
- 번식 휴지기가 끝난 암수가 같은 수조에 존재한 채 3초 동안 조건을 유지하면 75초의 포란을 시작한다. 현재 MVP는 우연한 이동 경로가 번식 자체를 막지 않도록 교미 거리 조건을 두지 않는다.
- 포란은 에너지 `0.42` 이상이고 최근 20초 안에 먹은 동안에만 진행된다. 조건이 나쁘면 느려지는 대신 진행이 멈춘다.
- 출산 시 결정적 시드에 따라 1~2마리의 어린 개체가 태어난다.
- 출산 뒤 암컷은 160초, 참여 수컷은 45초의 번식 휴지기를 가지며, 새로 성체가 된 개체는 120초가 지나야 번식할 수 있다.
- 개체별 수명은 900~1350 시뮬레이션초이며, 도달 시 에너지와 무관하게 자연사한다.
- 정상 플레이에서는 먹이가 충분하고 출생이 사망보다 많으면 개체군이 계속 증가한다. 먹이 소비가 생산을 넘으면 성장·포란이 멈추고 굶주림과 자연사로 감소한다.
- 초기 지급 성체에도 초기 번식 휴지기를 적용해 수조를 구성하자마자 연속 출산하지 않게 한다.

번식 가능 개체 수를 배치 개수, 먹이 셀 수 또는 순간 생체량에서 계산한 숨은 수용력으로 막지 않는다. 개체 수는 에너지, 최근 실제 섭식 여부, 암수 존재, 번식 휴지기, 먹이 고갈과 사망의 결과로 나타난다. 따라서 광량과 서식 표면을 개선해 조류 생산량을 늘리면 더 많은 새우를 부양할 수 있지만, 그 관계를 별도의 개체 수 공식으로 환산하지 않는다.

단, 손상된 저장 상태나 극단적인 장기 실행에서 한 번에 무한한 개체가 할당되는 것을 막기 위해 `2,048`마리에 마지막 부화 안전선을 둔다. 이 값은 정상 플레이의 생태 평형이나 번식 확률에 관여하지 않는 기술 가드다.

이 값들은 현실 시간을 그대로 재현하는 수치가 아니라, 먹이 공급과 개체군 변화가 한 미션 안에서 관찰되도록 압축한 게임 밸런스다.

### 24.5 생지화학 ledger의 현재 경계

현재 시뮬레이션은 `36 × 20` 공간 ledger에 다음 잠재 흐름을 기록한다.

- 조류의 잠재 산소 생산
- 조류와 체리새우의 잠재 산소 요구
- 실제 섭식량에 따른 용존 배설물 발생
- 사망 위치의 고형 유기물 질량

`effectsEnabled`는 현재 항상 `false`다. ledger는 후속 산소·오염·분해 시스템을 연결하기 위한 입력 기록일 뿐, 용존산소나 오염 농도를 만들지 않으며 조류·새우의 건강, 성장, 번식, 사망과 미션 판정에 되먹임하지 않는다. 일반 UI에도 아직 표시하지 않는다.

새우가 굶거나 개체 수명에 도달해 죽으면 생존 개체와 미션 성체 수에서 즉시 제외되고 사망 위치의 고형 유기물량을 ledger에 한 번 더한다. 동시에 위치·방향·몸길이·생활 단계를 보존한 사체 스냅샷을 만들며, 기존 새우 리그를 창백하고 움직임이 멈춘 말린 자세로 재사용한다. 사체는 55 시뮬레이션초 동안 남아 조금 가라앉고 마지막 구간에서 흐려진다. 사체를 선택하면 먹이 부족 또는 노화 자연사와 사망 후 경과시간을 확인할 수 있다. 박테리아 분해, 산소 고갈과 수질 피해는 아직 구현하지 않는다.

### 24.6 실험실 지원

실험실에서는 세 종류 구조물, 두 조류와 체리새우를 무제한으로 꺼낼 수 있다. 도전 모드와 동일한 광량·조류 성장·섭식·에너지·성장·번식·사망 규칙을 사용하고, 목표나 제한시간은 적용하지 않는다. 실행 후 배치를 바꾸려면 먼저 일시정지해야 한다.

개별 체리새우를 선택하면 생활 단계, 성별, 현재 행동, 에너지, 번식 상태와 실제 누적 섭식량을 확인할 수 있다. 영역 선택은 해당 영역의 성체·어린 개체 수, 평균 에너지와 누적 섭식량을 집계한다. 사체는 개별 선택으로 사망 원인과 경과시간을 확인하고, 관찰 패널은 사체가 있을 때만 죽은 새우 수를 표시한다. 관찰 패널에는 수조 전체 누적 섭식량과 전체 조류량·체리새우 개체 수의 시간 추세도 함께 표시한다. 산소·배설·사망 질량 ledger의 원시 수치는 실험실에서도 아직 공개하지 않는다.

### 24.7 추가 수용 기준

- [ ] 새우가 먹은 셀의 조류가 실제 섭취량만큼 감소한다.
- [ ] 같은 셀을 여러 마리가 먹어도 남은 조류보다 많은 양을 섭취하지 않는다.
- [ ] 배부른 새우는 먹이 목표를 놓고 탐색·휴식하며 모든 개체가 군락에 계속 붙어 있지 않는다.
- [ ] 개체별·전체 누적 섭식량과 조류량 추세가 실제 생체량 감소와 일치한다.
- [ ] 먹이 없는 기본 성체 4마리는 120초 유지 전에 에너지가 고갈되어 미션에 실패한다.
- [ ] 충분한 조류가 있는 구성은 성체 4마리를 연속 120초 유지할 수 있다.
- [ ] 어린 개체는 180초의 나이·급식 성장 조건을 충족하기 전에는 성체 목표에 포함되지 않는다.
- [x] 암수·최근 8초 내 실제 섭식·에너지·휴지기 조건이 맞으면 포란과 1~2마리 출산이 발생한다.
- [x] 생태 개체 수 상한 없이 먹이가 충분하면 출생이 계속되고, 소비가 생산을 넘으면 번식 중단·굶주림·사망으로 개체군이 감소한다.
- [x] 조류의 이웃 정착은 출발 셀에서 차감한 생체량만 도착 셀에 더하며, 확산 단계 전후의 전체 생체량이 보존된다.
- [x] `2,048`마리 기술 안전선 아래에서는 개체 수 자체가 번식 조건을 바꾸지 않는다.
- [ ] 개체 사망은 ledger의 고형 유기물량을 한 번만 늘리고, 생존 개체와 구분되는 사체 자세·사망 원인·경과시간을 표시하지만 아직 수질 피해는 만들지 않는다.
- [ ] `effectsEnabled = false`에서 ledger 값을 바꿔도 생물 상태와 미션 결과가 달라지지 않는다.
- [ ] 실험실에서 체리새우를 무제한 배치하고 같은 섭식·생애 주기를 관찰할 수 있다.
